## 需求文档：ProClaw（Agent 化架构）PRD v6.0

### 1. 背景与目标

#### 1.1 背景
- ProClaw 原版专注于小商户的进销存 + 协同办公，功能垂直且完整。
- 部分用户（如自由职业者、咨询公司、项目制团队、虚拟组织）不需要进销存，但需要公司管理工具（财务、任务、文档、客户关系等）。
- 传统 SaaS 一体化软件功能固定，难以满足不同虚拟团队个性化需求。
- Agent 架构兴起，通过可插拔的智能模块来按需组装业务能力，成为趋势。

#### 1.2 目标
- 推出 **ProClaw 虚拟公司简化版**：剥离进销存模块，原版改为**ProClaw 虚拟公司进销存版**。
- ProClaw升级为 **Agent 化架构**，必装 **财务管理 Agent**和 **营销管理 AiTeam**，并支持从Agent 市场安装其他能力（如任务管理、客户关系、文档协作等）。
- 提供 **Agent 市场**（`nvwa.proclaw.cc`），允许第三方开发者上架 Agent，用户一键安装启用。
- 保留原有桌面端 + 移动端基础框架（聊天、文件、设备直连），但业务模块全部由 Agent 提供。
- 财务管理 Agent 必须预装且不可卸载，保障基础财务记账、报表、收付款功能。

---

### 2. 用户故事

- **作为** 虚拟公司老板，我安装 ProClaw 虚拟公司后，自动拥有财务管理 Agent，可以记录收入支出、生成报表、管理发票。我还可以通过“招募控件”浏览 Agent 市场，安装任务管理 Agent，让团队成员分配任务。
- **作为** 团队成员，我可以使用已安装的 Agent 进行协作（例如在任务 Agent 中领取任务，在财务 Agent 中提交报销）。
- **作为** 第三方开发者，我可以将开发的 Agent 上传到 `nvwa.proclaw.cc`，设定价格或免费，供用户安装。

### 3. 技术架构

#### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│              ProClaw 桌面端                                  │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  Tauri 2 + React (核心框架)                              │ │
│  │  - 聊天、文件、设备授权（复用现有）                       │ │
│  │  - Agent 运行时容器 (隔离的 WebWorker 或 iframe)         │ │
│  │  - Agent 管理界面（招募控件）                            │ │
│  └─────────────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  必装 Agent：财务管理 Agent (内置)                       │ │
│  └─────────────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  可选 Agent：从市场安装 (如任务、CRM、文档)              │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                            │
                            │ HTTPS / WebSocket (直连/中继)
                            ▼
┌─────────────────────────────────────────────────────────────┐
│              移动端 App (React Native)                       │
│  - 基础框架 + Agent 视图容器                                 │
│  - 与桌面端同步 Agent 状态                                   │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│         Agent 市场 (nvwa.proclaw.cc)                        │
│  - Agent 列表、详情、下载、版本管理                          │
│  - 开发者上传、审核、计费接口                               │
└─────────────────────────────────────────────────────────────┘
```

#### 3.2 Agent 运行时设计

- **隔离性**：每个 Agent 运行在独立的 WebWorker 或 iframe 中，与主框架隔离，避免相互影响。
- **API 桥接**：Agent 通过 `postMessage` 与主框架通信，主框架提供统一的 API（获取用户信息、读写本地数据库、发送聊天消息、调用支付等）。
- **权限管理**：安装 Agent 时，需声明所需权限（如读取财务数据、发送消息、访问文件），用户确认后才可启用。
- **数据存储**：每个 Agent 拥有独立的本地存储空间（IndexedDB 或 SQLite 表前缀），主框架不干预。
- **生命周期**：Agent 可以被启用、禁用、卸载。禁用后不占用资源，卸载后清除所有数据。

#### 3.3 Agent 市场集成

- 桌面端内置“招募控件”按钮，打开 Agent 市场 WebView 或 iframe，展示 `nvwa.proclaw.cc`。
- 用户点击“安装”后，市场返回 Agent 包 URL（签名后的 zip 文件），桌面端下载、解压、校验签名，注册到 Agent 管理器中。
- Agent 更新：市场推送更新通知，用户确认后增量更新。

### 4. 功能需求

#### 4.1 桌面端核心功能

| 模块               | 功能                                                         |
| ------------------ | ------------------------------------------------------------ |
| **Agent 管理器**   | - 显示已安装 Agent 列表（图标、名称、版本、状态）。<br>- 支持启用/禁用/卸载/配置。<br>- 查看 Agent 权限详情。 |
| **招募控件**       | - 按钮“发现更多 Agent”，点击打开市场窗口。<br>- 展示市场推荐、分类、搜索。<br>- 支持一键安装（需用户确认权限）。 |
| **财务管理 Agent** | - 内置且不可卸载。<br>- 功能：记账（收入/支出/转账）、账户管理、预算、报表（月度/年度）、发票管理（可对接电子发票）。<br>- 数据模型复用原有进销存中的财务部分（应收/应付可简化）。 |
| **其他 Agent**     | - 任务管理 Agent（看板、任务分配、进度跟踪）。<br>- 客户关系 Agent（联系人、沟通记录、商机）。<br>- 文档管理 Agent（Markdown 编辑器、版本历史）。<br>- 等等，由市场提供。 |

#### 4.2 移动端功能

- 与桌面端功能对等：支持浏览已安装 Agent 的移动视图（Agent 需提供响应式界面）。
- 聊天和通知中可调用 Agent 快捷操作（如“创建任务”）。
- 通过 WebSocket 与桌面端同步 Agent 状态（如任务变化）。

#### 4.3 财务管理 Agent 详细功能（必装）

- **账户管理**：支持现金、银行账户、第三方支付账户，记录余额。
- **收支记录**：分类（餐饮、交通、办公等），支持附件上传（发票照片）。
- **预算控制**：按月/类别设置预算，超出预警。
- **报表**：收支明细表、分类统计图、现金流量表（简易版）。
- **发票管理**：记录进项/销项发票，支持扫码识别（接入现有 AI 识别能力）。
- **对账**：与银行流水导入（手动 CSV 或未来 API 对接）。
- **权限**：老板可查看所有，财务人员可操作，普通成员只能查看自己提交的报销。

#### 4.4 Agent 市场（nvwa.proclaw.cc）功能

- **开发者控制台**：注册、上传 Agent 包、填写描述、截图、定价。
- **审核机制**：官方审核确保安全（自动化扫描 + 人工抽检）。
- **用户端**：搜索、分类、评分、评论、安装量统计。
- **版本管理**：支持多版本发布，用户可选择回退。
- **计费**：支持免费、一次性付费、订阅制（对接支付网关）。

### 5. 数据模型（桌面端 SQLite 扩展）

#### 5.1 Agent 注册表

```sql
CREATE TABLE agents (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  version TEXT NOT NULL,
  manifest JSON NOT NULL,        -- 包含入口文件、权限声明、图标等
  enabled BOOLEAN DEFAULT 1,
  installed_at INTEGER,
  last_updated INTEGER
);
```

#### 5.2 Agent 数据存储约定

- 每个 Agent 使用独立的表前缀：`agent_<agent_id>_*`，主框架不主动访问。
- Agent 通过主框架提供的 API 读写共享数据（如用户信息、聊天消息、文件索引），需经用户授权。

#### 5.3 财务管理 Agent 数据表（内置）

```sql
-- 账户表
accounts (id, name, type, balance, currency, ...);

-- 交易记录表
transactions (id, account_id, amount, category, type, date, note, attachment_id, created_by);

-- 预算表
budgets (id, category, month, limit_amount, actual_amount);

-- 发票表
invoices (id, type, code, amount, tax, status, issue_date, file_id);
```

### 6. API 设计（主框架暴露给 Agent）

Agent 通过 `window.proclaw` 或 `postMessage` 调用：

| API 名称                     | 说明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| `proclaw.getCurrentUser()`   | 获取当前用户信息（姓名、角色、权限）。                        |
| `proclaw.db.query(sql, params)` | 执行只读 SQL 查询（仅限 Agent 自己的表）。                     |
| `proclaw.db.execute(sql, params)` | 执行写 SQL（需声明权限）。                                 |
| `proclaw.storage.get(key)`   | 读取 Agent 私有存储中的键值。                                 |
| `proclaw.storage.set(key, value)` | 写入私有存储。                                               |
| `proclaw.sendMessage(to, content)` | 通过聊天模块发送消息。                                       |
| `proclaw.showNotification(title, body)` | 显示系统通知。                                               |
| `proclaw.openAgentMarket()`   | 打开市场窗口。                                               |

### 7. Agent 打包与签名规范

- Agent 包为 zip 格式，包含：
  - `manifest.json`（名称、版本、入口文件、权限列表、图标）。
  - 前端资源（HTML/CSS/JS），入口文件需挂载到指定 div。
- 签名：开发者使用私钥对 `manifest.json` 签名，桌面端验证市场公钥。
- 安全限制：Agent 只能通过桥接 API 访问资源，禁止直接网络请求（可代理通过主框架）。

### 9. 与现有 ProClaw 的关系

- **简化版** 与 **进销存版** 共享相同的桌面端基础框架（聊天、设备直连、用户权限等）。
- 通过编译时特性标志（feature flag）移除进销存模块，添加 Agent 模块。
- 用户在下载时可选择“模式”（进销存模式/虚拟公司模式），分开发布。

---

### 10. 风险与备选

- **Agent 安全风险**：恶意 Agent 可能窃取数据。应对措施：权限声明 + 用户确认 + 代码沙箱（可选用 QuickJS 或 Wasm 隔离，初期 WebWorker 基本足够）。
- **市场生态冷启动**：初期内置几个官方 Agent（任务、CRM），并开放少量第三方内测。
- **性能问题**：多个 Agent 同时运行时可能占用资源。限制同时启用 Agent 数量（默认 10 个），并提供性能监控。
