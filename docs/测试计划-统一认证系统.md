## 一、测试范围

### 1.1 用户认证流程（核心）

| 模块 | 测试内容 |
|------|----------|
| **授权码流程** | Authorization Code Flow + PKCE 的完整流程 |
| **Token 交换** | 使用 authorization_code 换取 access_token/id_token/refresh_token |
| **Token 刷新** | 使用 refresh_token 刷新 access_token |
| **Token 验证** | id_token 签名验证、claims 校验（iss/aud/exp/nonce） |
| **用户信息获取** | 通过 access_token 调用 UserInfo 端点 |
| **登出** | 调用 logout 端点使 refresh_token 失效 |

### 1.2 权限管理

| 模块 | 测试内容 |
|------|----------|
| **Scope 控制** | openid、profile、email scope 的正确请求与响应 |
| **角色识别** | id_token 中 is_admin 字段的正确返回与应用端处理 |
| **访问控制** | 基于角色的页面/功能访问权限控制 |

### 1.3 单点登录（SSO）功能

| 模块 | 测试内容 |
|------|----------|
| **跨子域 SSO** | account.proclaw.cc 登录后，app.proclaw.cc 无需再次登录 |
| **Session 共享** | pc_session cookie 在 .proclaw.cc 域名下的跨应用共享 |
| **统一登出** | 一处登出后，所有关联应用均需重新登录 |

### 1.4 三端集成测试

| 平台 | 测试内容 |
|------|----------|
| **桌面端（Tauri）** | 通过系统浏览器启动 OIDC 流程，127.0.0.1:1410 回调接收 |
| **云在线版（Next.js）** | 浏览器内 OIDC 跳转，middleware 自动重定向，auth/callback 路由处理 |
| **移动端（React Native）** | 系统浏览器打开授权页面，深链 proclaw://oauth/callback 回调 |

### 1.5 安全测试

| 测试项 | 内容 |
|--------|------|
| **PKCE 验证** | 无 code_challenge 时拒绝授权；code_verifier 不匹配时拒绝 token |
| **CSRF 防护** | state 参数验证失败时拒绝处理 |
| **重放攻击防护** | nonce 参数验证 |
| **Token 安全性** | access_token/id_token 过期处理，refresh_token 轮换机制 |
| **HTTPS 强制** | 生产环境强制 HTTPS，HTTP 请求被拒绝或重定向 |

### 1.6 非功能测试

| 测试项 | 内容 |
|--------|------|
| **兼容性测试** | 不同浏览器、设备、操作系统的兼容 |
| **性能测试** | 授权响应时间、Token 交换性能、高并发场景 |
| **稳定性测试** | 长时间运行、网络异常恢复、服务故障恢复 |
| **可用性测试** | 错误提示友好性、操作流程直观性 |

```
步骤 3: 执行集成测试验证
```

### 4.6 验收测试阶段（T11）

```
步骤 1: 由产品团队执行验收测试
步骤 2: 确认所有需求已满足
步骤 3: 输出测试报告
```

## 五、预期结果与判定标准

### 5.1 通过标准

| 测试类型 | 通过条件 |
|----------|----------|
| **功能测试** | 所有 P0 级用例 100% 通过；P1 级用例 ≥ 95% 通过；P2 级用例 ≥ 80% 通过 |
| **安全测试** | 所有安全用例通过，无高危漏洞 |
| **兼容性测试** | 主流浏览器/设备 100% 通过 |
| **性能测试** | 响应时间指标达标，高并发场景无错误 |
| **集成测试** | 三端对接均正常工作 |

### 5.2 缺陷严重程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0 - 致命** | 导致系统崩溃、数据丢失或安全漏洞 | 登录功能完全不可用；Token 验证逻辑错误 |
| **P1 - 严重** | 核心功能无法正常使用，影响用户体验 | Token 刷新失败；SSO 不工作 |
| **P2 - 中等** | 非核心功能问题，有替代方案 | 错误提示信息不友好；特定浏览器样式问题 |
| **P3 - 轻微** | 不影响功能的小问题 | 界面文字拼写错误；按钮样式不一致 |

### 5.3 上线条件

- ✅ 所有 P0 级缺陷已修复
- ✅ 所有安全测试用例通过
- ✅ 三端集成测试通过
- ✅ 性能指标达标
- ✅ 测试覆盖率 ≥ 80%（核心模块）

## 六、缺陷管理流程

### 6.1 缺陷发现

```
测试人员发现缺陷 → 记录缺陷详情（重现步骤、预期结果、实际结果、截图/日志）
```

### 6.2 缺陷提交

```
提交至缺陷管理系统（如 Jira）：
- 缺陷标题：清晰描述问题
- 缺陷描述：详细重现步骤
- 严重程度：P0/P1/P2/P3
- 所属模块：认证流程/权限管理/SSO/三端集成
- 测试环境：测试环境/开发环境
- 附件：截图、日志、HAR 文件
```

### 6.3 缺陷处理

```
缺陷分配 → 开发修复 → 测试验证 → 关闭/重新打开

处理时限：
- P0：24 小时内修复
- P1：3 个工作日内修复
- P2：1 周内修复
- P3：下个迭代修复
```

### 6.4 缺陷跟踪

- 每日站会同步缺陷状态
- 每周输出缺陷统计报告
- 确保缺陷闭环率 ≥ 95%

## 九、风险评估与应对措施

### 9.1 技术风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| OIDC Provider 服务不稳定 | 中 | 高 | 部署多节点、监控告警、故障转移 |
| Token 验证逻辑错误 | 低 | 高 | 代码审查、单元测试覆盖、模拟测试 |
| PKCE 实现不一致 | 低 | 高 | 统一 PKCE 工具库、跨平台测试 |
| 深链回调在某些设备不工作 | 中 | 中 | 多设备测试、备选回调方案 |
| 跨域问题 | 中 | 中 | 配置正确的 CORS、使用子域共享 cookie |

### 9.2 环境风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 测试环境与生产环境差异 | 高 | 中 | 尽量保持环境一致、配置自动化 |
| 测试账号数据不一致 | 中 | 中 | 自动化数据初始化、定期清理 |
| 网络波动影响测试 | 中 | 中 | 网络监控、重试机制、离线测试 |

### 9.3 进度风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 缺陷修复周期过长 | 中 | 高 | 提前沟通、优先级排序、资源预留 |
| 兼容性问题发现较晚 | 中 | 中 | 提前进行兼容性测试、使用设备云 |
| 安全漏洞影响上线 | 低 | 高 | 提前安全审计、预留修复时间 |

### 9.4 应急预案

1. **认证中心服务故障**：切换至备用节点，启用缓存的用户信息
2. **Token 服务不可用**：保留本地 session，待服务恢复后同步
3. **网络完全中断**：提供离线模式，网络恢复后自动同步
4. **数据损坏**：定期备份，快速恢复
