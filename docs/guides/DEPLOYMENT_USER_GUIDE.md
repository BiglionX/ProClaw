## 2. 快速部署

### 2.1 Windows 安装

1. 下载最新安装包 `ProClaw_Setup_x64.msi`
2. 双击运行安装程序
3. 按照向导完成安装
4. 首次启动时会自动初始化数据库

### 2.2 从源码构建

**前置条件**：
- Node.js 18+
- Rust 1.75+
- Tauri CLI
- WiX Toolset（Windows 打包）

```bash
# 克隆仓库
git clone https://github.com/proclaw/ProClaw.git
cd ProClaw

# 安装依赖
npm install

# 启动开发模式
npm run tauri dev

# 构建安装包
npm run tauri build
```

详细构建脚本：`build-release.ps1`（Windows PowerShell）

### 2.3 首次启动

1. 双击 ProClaw 图标启动应用
2. 系统自动初始化 SQLite 数据库
3. 默认管理员账号：`boss` / `IamBigBoss`
4. 建议首次登录后修改密码

### 2.4 局域网配置

1. 确保桌面端防火墙允许 8888 端口入站
2. 查看本机 IP 地址：`ipconfig`（Windows）
3. 在应用设置中查看显示的 IP 地址

**推荐使用 Tailscale** 实现安全局域网连接：
```bash
# 安装 Tailscale
# 在所有设备上安装并登录同一账号
# 使用 Tailscale IP 进行连接
```

## 3. Pro 版开通

### 3.1 Pro 版功能

| 功能 | 社区版 | Pro 版 |
|------|--------|--------|
| 基础进销存 | ✅ | ✅ |
| 局域网移动端连接 | ✅ | ✅ |
| AI 订单识别 | ❌ | ✅ |
| 离线队列（7天） | ❌ | ✅ |
| 云中继故障转移 | ❌ | ✅ |
| 端到端加密云备份 | ❌ | ✅ |
| 推送通知 | ❌ | ✅ |

### 3.2 开通步骤

1. 打开 ProClaw 桌面端
2. 进入「设置」→「订阅管理」
3. 选择套餐：
   - 基础版：¥9.9/月（1 万 token）
   - 专业版：¥39.9/月（10 万 token）
   - 企业版：¥99/月（100 万 token）
4. 完成支付
5. 自动激活 Pro 功能

### 3.3 Supabase 配置

Pro 版需要配置 Supabase：

1. 注册 [Supabase](https://supabase.com) 账号
2. 创建项目
3. 获取 `URL` 和 `anon key`
4. 在 ProClaw 设置中填入 Supabase 凭证
5. 运行数据库迁移脚本：`database/supabase_migrations.sql`

### 3.4 AI 模型配置

ProClaw 支持多种 AI 模型：

| 模型 | 接入方式 | 费用 |
|------|----------|------|
| ProClaw 集成 LLM | 内置（默认） | 5 token/次 |
| OpenAI GPT-4V | 自备 API Key | 按 API 实际消耗 |
| 阿里云通义千问 | 自备 API Key | 按 API 实际消耗 |
| 智谱清言 GLM-4 | 自备 API Key | 按 API 实际消耗 |
| 本地 PaddleOCR | 无需配置 | 免费 |

进入「设置」→「AI 配置」添加提供商。

## 4. 移动端配对

### 4.1 配对流程

1. 桌面端进入「设置」→「设备管理」
2. 点击「生成配对码」
3. 桌面端显示二维码和配对码（5 分钟有效）
4. 移动端打开 ProClaw App
5. 扫描二维码或输入配对码
6. 连接成功后自动获取访问 token

### 4.2 连接模式

| 模式 | 使用场景 | 配置要求 |
|------|----------|----------|
| 直连 | 同一局域网 | 桌面端 IP 可达 |
| Tailscale | 任意网络 | 安装 Tailscale |
| 云中继 (Pro) | 直连失败 | Supabase 已配置 |
| 离线 | 无网络 | 数据本地缓存 |

### 4.3 设备管理

- 桌面端可查看已授权设备列表
- 支持远程踢除可疑设备
- token 过期后需重新配对

## 5. 常用功能指南

### 5.1 商品管理

**简单商品库模式**：
1. 进入「商品库」
2. 点击「添加产品」
3. 填写 SKU、名称、成本价、售价
4. SKU 支持自动生成

**电商商品库模式（SPU-SKU）**：
1. 在简单模式下点击「升级为电商商品库」
2. 系统自动迁移数据创建 SPU 和默认 SKU
3. 支持多规格、多图管理
4. 可通过「返回简单模式」降级（⚠️ 数据会丢失）

### 5.2 销售流程

1. 进入「销售」→「创建订单」
2. 选择客户 → 添加产品 → 填写数量/价格
3. 可选：AI 识别手写订单图片生成订单
4. 确认订单 → 自动扣减库存 → 生成订单号
5. 可查看订单状态、打印、导出

### 5.3 AI 订单识别

1. 进入「销售」→「AI 识别订单」
2. 拍照或上传手写/打印订单图片
3. 系统自动识别商品行、数量、单价
4. 人工修正识别结果
5. 点击「AI 审核」校验库存和价格
6. 确认生成销售单

### 5.4 采购管理

1. 先添加供应商（「采购」→「供应商管理」→「添加供应商」）
2. 创建采购订单：选择供应商 → 添加产品
3. 确认后等待收货
4. 收货时「确认入库」自动增加库存

### 5.5 库存盘点

1. 进入「库存」→「盘点」
2. 创建盘点单
3. 录入实际库存数量
4. 系统自动生成差异调整记录

### 5.6 权限管理

**角色列表**：
| 角色 | 权限范围 |
|------|----------|
| 老板 (boss) | 全部权限 |
| 财务 (finance) | 财务管理、产品查看 |
| 采购 (purchase) | 采购管理、库存查看 |
| 仓库 (warehouse) | 库存管理、产品查看 |
| 销售 (sales) | 销售管理、库存查看 |
| 客户 (customer) | 查看自己的订单 |
| 供应商 (supplier) | 查看自己的产品 |

进入「设置」→「用户管理」分配角色。

## 6. 故障排除

### 6.1 桌面端无法启动

1. **检查端口占用**：8888 端口是否被其他程序占用
2. **数据库错误**：删除 `%APPDATA%\com.proclaw.desktop\proclaw.db` 重新初始化
3. **缺少运行时**：安装 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)

### 6.2 移动端无法连接

1. 确认桌面端和移动端在同一网络
2. 检查桌面端防火墙是否开放 8888 端口
3. 尝试使用 Tailscale 连接
4. 重新生成配对码

### 6.3 AI 识别失败

1. 确认 Pro 版订阅有效
2. 检查网络连接
3. 尝试降低图片大小（建议 < 5MB）
4. 确保图片清晰、光线充足

### 6.4 数据同步问题

1. 检查 Supabase 配置是否正确
2. 查看「设置」→「同步状态」
3. 手动触发同步
4. 检查网络连接

## 7. 数据备份与恢复

### 7.1 本地备份

```bash
# 数据库文件位置
%APPDATA%\com.proclaw.desktop\proclaw.db

# 复制备份
copy proclaw.db proclaw_backup_20260101.db
```

### 7.2 Pro 版云备份

1. 进入「设置」→「数据管理」
2. 开启「自动云备份」
3. 设置备份频率（每日/每周）
4. 数据自动加密后上传至 Supabase

### 7.3 数据恢复

1. 关闭 ProClaw
2. 将备份文件覆盖 `proclaw.db`
3. 重新启动 ProClaw

## 8. 安全最佳实践

1. **修改默认密码**：首次登录后立即修改 boss 账号密码
2. **使用 Tailscale**：推荐通过 Tailscale 连接移动端，自动加密传输
3. **定期备份**：设置自动备份策略
4. **设备管理**：定期审查已授权设备，踢除不认识的设备
5. **防火墙**：仅在局域网开放 8888 端口，不要直接暴露到公网
6. **API Key 安全**：不要在公开渠道分享你的 AI API Key
7. **系统更新**：保持 ProClaw 更新到最新版本
