## 需求文档：云商城 AI 客服模块（独立客服 Agent，仅复用秘书 Agent 头像）

**版本**：v1.0  
**说明**：AI 客服与秘书 Agent 定位不同，**仅头像相同**，能力、数据、逻辑完全独立。  
**依赖**：ProClaw 云商城（PRD v5.0）、ProClaw 虚拟公司版（PRD v6.x，仅提供头像资源）  
**目标**：为云商城提供独立的智能客服 Agent，支持聊天记录查询、转人工、自动学习。

---

### 1. 核心区别说明

| 角色 | 云商城 AI 客服 Agent |
|------|---------------------|
| **服务对象** | 商城访客（外部客户） |
| **职责** | 回答商品咨询、订单问题、售后政策 |
| **数据来源** | 商品库、知识库（FAQ）、订单状态（只读） |
| **转人工** |  支持转老板或运营 |
| **聊天记录存储** | 云端数据库（多租户） |
| **是否可用大模型** | 是（生成回答，但受限于知识库） |

**仅共用**：头像图片 URL（商户可自定义，默认使用秘书头像样式，但两者独立配置）。

---

### 2. 背景与目标

#### 2.1 背景
- 云商城需要客服功能，但商户不希望客户直接联系老板私人微信。
- ProClaw 已有成熟的 Agent 体系，但秘书 Agent 不适合对外。
- 需要一个独立、轻量的 AI 客服 Agent，可复用 ProClaw 的品牌形象（头像），但逻辑独立。

#### 2.2 目标
- 云商城右下角提供 AI 客服，使用与秘书 Agent 相同风格的头像（但商户可更换）。
- 客服 Agent 的知识库独立于秘书 Agent，仅包含商品信息、常见问答、订单查询接口。
- 当无法回答时，支持转人工（老板或运营在 ProClaw 桌面端回复）。
- 所有聊天记录存储在云端，商户可在桌面端查看、管理问答库。

---

### 3. 功能需求

#### 3.1 云商城前端（客户侧）

- 右下角悬浮按钮，**头像** 默认与秘书 Agent 相同（但代码中为独立配置，可后期调整）。
- 点击打开聊天窗口，窗口标题为“智能客服”（不是“秘书”）。
- 自动问候：“您好，我是客服小如，请问有什么可以帮您？”

#### 3.2 AI 客服后端（独立服务）

**知识库**（需商户在桌面端维护）：
- 商品库：自动同步商城上架商品信息（名称、价格、规格、库存）。
- 问答库：商户可手动添加或从转人工中保存得到（问题 → 标准答案）。
- 订单查询：允许客户输入订单号，客服 Agent 调用订单 API 返回状态（不暴露敏感信息）。

**处理流程**：
1. 客户问题 → 先检索商品库（关键词匹配 + 向量相似度）。
2. 若未命中，检索问答库（精确匹配 + 向量）。
3. 若仍未命中，检查是否为订单查询（“我的订单……”→ 引导输入订单号）。
4. 如果都不是，则根据商户配置的模式（A/B）决定是否转人工。

#### 3.3 转人工模式（可配置）

| 模式 | 行为 |
|------|------|
| **A：直接转人工** | 客服回复：“您的问题我将转给人工客服，请稍后。”同时在 ProClaw 桌面端生成提醒。老板回复后自动推送给客户。 |
| **B：大模型判断后转** | 调用大模型评估问题是否需要人工（如主观评价、议价、非标需求）。若需要，同模式A；否则大模型生成通用回答（如“建议您查看商品详情页”），但不保存到问答库。 |

#### 3.4 商户端管理（ProClaw 桌面端）

**入口**：云商城运营 AI Team → “AI 客服” 子模块（独立于秘书 Agent 界面）

**功能**：
- **历史聊天记录**：查看所有客户与 AI 客服的对话，可搜索、导出。
- **未处理转人工请求**：老板或运营可在此回复，回复后自动发送给客户，并询问“是否保存此问答到问答库？”（弹出确认框）
- **问答库管理**：独立于秘书的知识库，仅用于客服。
- **客服设置**：选择转人工模式、设置自动回复阈值、上传客服头像（默认秘书头像样式，可替换）。

#### 3.5 聊天记录存储

- 云端表 `customer_service_chat_logs`（独立于秘书的对话表）。
- 字段：`session_id`, `customer_id`, `question`, `answer`, `answer_source`（`knowledge_base`/`model`/`manual`）, `is_manual`（是否转人工）, `created_at`。

### 4. 与现有系统的集成点

| 集成点 | 说明 |
|--------|------|
| **头像资源** | 秘书 Agent 的头像存储在 ProClaw 桌面端的 `assets/avatar_secretary.png`。AI 客服默认复制该文件到独立目录，商户可覆盖。 |
| **商品库** | 复用云商城的商品快照表（`products_snapshot`），只读访问。 |
| **订单 API** | 调用云商城的订单查询接口（需商户授权）。 |
| **桌面端通知** | 当有转人工请求时，通过 WebSocket 推送消息到 ProClaw 桌面端，显示红点提醒。 |

### 5. UI 原型（文字描述）

**云商城端**：
- 右下角圆形按钮，显示客服头像（默认秘书风格）。
- 窗口宽 360px，高 500px，支持拖拽。
- 消息气泡：客户右灰色，客服左绿色。
- 输入框支持文字、表情。

**ProClaw 桌面端**：
- 左侧菜单-联系人-增加“AI客服小如”图标-带头像哦（boss可以在这里设置客服头像和昵称）。
- 左侧菜单-aiteam-云商城运营小组-详情-云客服-详情页主区域分为三个标签页：**待回复**（红点）、**聊天记录**-快捷控件-**问答库**。
- 左侧菜单-消息-AI客服小如：老板，这个问题怎么回？待回复列表中每条显示客户问题、客户 ID、时间。
- boss在对话框，填写答案后发送，并询问“是否保存到问答库？”（是/否/不再询问）。
