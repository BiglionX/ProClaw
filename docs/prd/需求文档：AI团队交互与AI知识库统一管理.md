## 一、背景与问题

### 1.1 当前问题

| 问题 | 描述 |
|------|------|
| AI 团队"只可看不可聊" | Dashboard 的 AI Team 状态卡片和 TeamsPage 只能看到团队存在，Boss 无法与团队（或团队成员）进行对话、下达指令、查看工作成果 |
| 三库分散管理 | 媒体库、问答库、资料库分别占据三个独立菜单和页面，功能边界模糊（都是"AI 可用的知识资源"），商户理解成本高 |
| 跨版本不一致 | 三库目前只在 Light 版中可见，进销存版和虚拟公司版完全没有 |

### 1.2 目标

1. **AI 团队可交互**：Boss 可以点击团队 → 进入团队专属对话 → 下达指令 → 看到团队返回的工作成果
2. **AI 知识库统一**：将媒体库、问答库、资料库合并为一个「AI 知识库」菜单 + 一个分列式页面，全版本通用

## 二、功能一：AI 团队交互系统（全版本）

### 2.1 概述

为所有 ProClaw 版本的 AI 团队建立 **"点击即聊"** 的交互机制，使 Boss 能像和真人团队沟通一样向 AI 团队下达指令、查看工作进度和输出成果。

### 2.2 交互入口

| 入口 | 触发方式 | 行为 |
|------|---------|------|
| **Dashboard AI Team 卡片 -「查看详情」** | 点击 | 打开 FloatingAgentChat，自动带入该团队的上下文（团队名称 + 角色），发起团队专属对话 |
| **Dashboard AI Team 卡片 -「配置」** | 点击 | 打开团队参数配置对话框（发布平台、关键词、定时任务等） |
| **TeamsPage 团队卡片 -「对话」** | 新增按钮 | 打开 FloatingAgentChat，定位到该团队的某个成员 |
| **TeamsPage 团队详情 - 成员列表** | 点击成员 | 打开 FloatingAgentChat，定位到该成员的专属对话 |

### 2.3 FloatingAgentChat 团队上下文增强

```
┌──────────────────────────────────────────┐
│  🤖 新媒体运营团队                    [─] │  ← 团队名称（上下文标识）
│  ─────────────────────────────────────── │
│  [快讯]  [生图]  [文案]  [数据分析]      │  ← 团队快捷指令
│  ─────────────────────────────────────── │
│                                          │
│  🧑 Boss: 帮我去小红书写一篇洗发水种草   │
│                                          │
│  🤖 新媒体运营:                           │
│  已为您生成小红书文案草稿：               │
│  ┌─────────────────────────────────┐    │
│  │ 📝 小红书种草文案 v1            │    │
│  │ 标题：秃头救星！这瓶洗发水...    │    │
│  │ [采纳] [修改] [换风格]          │    │  ← 工作成果卡片
│  └─────────────────────────────────┘    │
│  📎 关联：媒体库 > 产品图_洗发水.png    │
│  📎 关联：资料库 > 成分说明.pdf         │

│  ─────────────────────────────────────── │
│  💬 输入你的指令...                 [➤] │
└──────────────────────────────────────────┘
```

**关键特性**：
- `teamContext` 状态管理：记录当前对话的团队上下文（teamId, teamName, memberRole）
- 快捷指令随团队切换而变化（新媒体→生图文案、团购→订单查询、商城→商品同步）
- AI 回复中嵌入**工作成果卡片**（文案、图片、分析报告），Boss 可一键采纳

**工作日志数据模型**（新增）：
```typescript
interface TeamWorkLog {
  id: string;
  team_id: string;
  agent_id: string;
  agent_role: string;
  action: string;           // 动作描述
  result: string;           // 结果摘要

  status: 'completed' | 'in_progress' | 'failed';
  created_at: string;
}
```

### 3.3 布局规范

- **左侧面板**（200px）：垂直 Tab/列表，三个图标+名称+计数
  - 📷 媒体库 (32)
  - ❓ 问答库 (156)
  - 📄 资料库 (48)
- **右侧面板**（自适应）：根据选中 Tab 渲染对应内容（复用现有三个页面的组件逻辑）
- 三个模块的**数据层**（service）保持不变，仅 UI 层统一
