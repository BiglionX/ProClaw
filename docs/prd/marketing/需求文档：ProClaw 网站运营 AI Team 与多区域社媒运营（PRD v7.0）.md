### 1. 背景与目标

#### 1.1 背景
- ProClaw.cc 是 ProClaw 项目的官方网站，需要持续的内容更新、SEO 优化、用户增长和品牌推广。
- ProClaw 已与 Nuwax（Agent 市场 `nvwa.proclaw.cc`）打通，用户可以通过 Nuwax 搜索、创建个性化 AI Team。
- 用户希望先在自己的 ProClaw 实例中装入一个 **网站运营 AI Team**，实现官网的智能化自运营。
- 目标市场覆盖 **欧美、东南亚、国内**，不同市场需要差异化的社媒运营策略和内容风格，因此至少需要两组（实际为三组）社媒运营 AI Team。

#### 1.2 目标
- 为 ProClaw 构建一套可复用的 **网站运营 AI Team** 模板，包含 SEO、内容、数据分析、转化优化等 Agent。
- 内置 **多区域社媒运营 AI Team**（欧美、东南亚、国内各一组），每组负责对应市场的主流社交平台（如 Twitter/X、Facebook、Instagram、TikTok、微信/小红书等）。
- 支持通过 Nuwax 一键安装、配置，并与 ProClaw 官网无缝集成。
- 实现内容自动生成、定时发布、互动回复、广告效果追踪，最终提升官网流量和用户转化。

### 2. 用户故事

- **作为** ProClaw 项目负责人，我在 Nuwax 市场搜索“网站运营 AI Team”，点击安装。系统自动创建一组 Agent（SEO 专家、内容编辑、数据分析师、转化优化师），并绑定我的 ProClaw.cc 网站分析账户。AI Team 开始工作：每周生成 3 篇 SEO 友好的博客，监控关键词排名，自动优化页面标题和元描述。
- **作为** 市场负责人，我在同一 ProClaw 实例中安装“欧美社媒运营 Team”，授权 Twitter、Facebook 账号。该 Team 自动根据官网最新博客生成推文，每天发布 2 条，并回复用户提及。同时生成周报，显示曝光、点击、互动率。
- **作为** 东南亚市场专员，我安装“东南亚社媒运营 Team”，支持 TikTok 和 Instagram，内容使用当地语言（泰语、印尼语等），并自动追踪热点话题。
- **作为** 国内运营，我安装“国内社媒运营 Team”，支持微信公众号、小红书、知乎。AI Team 将官网技术文章改写为更适合国内平台的口吻，定时推送，并收集评论。

#### 3.2 关键组件
- **网站运营核心 Agent**：运行在 ProClaw 桌面端，通过调度任务（如定时爬取网站分析数据、生成内容、发送到社媒 API）实现运营自动化。
- **社媒 Agent**：每个 Agent 负责一个社交平台，调用平台官方 API 完成发布、互动、数据分析。支持多账号轮换。
- **区域适配层**：针对不同市场，内容生成 Agent 使用不同的 prompt 模板（语言、文化参照、热门话题库）。
- **Nuwax 模板**：提供一键安装包，包含所有 Agent 的 manifest、默认配置、依赖声明。

### 4. 功能需求

#### 4.1 网站运营 AI Team

| Agent 名称       | 功能                                                         |
| ---------------- | ------------------------------------------------------------ |
| **SEO Agent**    | - 监控官网关键词排名（通过 API 拉取 Google Search Console 数据）。<br>- 自动建议优化标题、元描述、内部链接。<br>- 生成 sitemap 并提交。<br>- 检测死链和加载速度问题。 |
| **内容生成 Agent** | - 根据 SEO 关键词列表，每周生成 3-5 篇技术博客或产品更新文章（中英文）。<br>- 使用 DeepSeek 或其他大模型，确保高质量、无幻觉。<br>- 生成后推送给 Boss 确认（确认卡片），通过后自动发布到官网（通过 CMS API）。 |
| **数据分析 Agent** | - 整合 Google Analytics 4 数据，生成日报/周报（流量来源、用户行为、转化率）。<br>- 发现异常趋势（如某页面跳出率升高）并发出预警。 |
| **转化优化 Agent** | - 分析用户行为漏斗，建议 A/B 测试方案。<br>- 自动调整网站弹窗、CTA 按钮等（需网站集成支持）。 |

#### 4.2 社媒运营 AI Team（每个区域独立）

**通用能力**：
- 绑定该区域的社交账号（每个平台至少一个）。
- 根据官网最新内容，自动生成适配该区域语言和文化的帖子（文字+图片/视频）。
- 定时发布（可设置频率，如每天 2-3 条）。
- 自动回复用户的提及和私信（使用大模型生成回复，需 Boss 审核模式或全自动）。
- 追踪互动数据（点赞、评论、转发），生成报表。
- 支持广告投放（可选，对接平台广告 API）。

**区域特殊要求**：

| 区域   | 平台                             | 语言与文化要求                                               |
| ------ | -------------------------------- | ------------------------------------------------------------ |
| 欧美   | Twitter, Facebook, Instagram, LinkedIn | 英语（美式/英式），注重专业性和技术讨论，发布时间参考美西/美东时区。 |
| 东南亚 | TikTok, Instagram, Facebook      | 泰语、印尼语、越南语等，内容短小、生动，利用当地热点和网红联动。 |
| 国内   | 微信公众号、小红书、知乎、微博    | 简体中文，风格需符合国内平台规范，可使用网络热词，注意合规审查。 |

#### 4.3 运营仪表板

- 在 ProClaw 桌面端增加“运营中心”视图，展示：
  - 网站 SEO 健康度评分
  - 各社媒账号的粉丝增长/互动趋势图
  - 待 Boss 确认的待发布内容列表
  - 异常预警列表
- 支持手动干预：暂停某个 Agent、修改待发布内容、直接发送消息。

### 5. 数据模型（SQLite 扩展）

#### 5.1 社媒账号绑定表

```sql
CREATE TABLE social_accounts (
  id TEXT PRIMARY KEY,
  region TEXT,                  -- 'us_eu', 'sea', 'cn'
  platform TEXT,                -- 'twitter', 'facebook', 'tiktok', 'wechat', etc.
  account_name TEXT,
  access_token_encrypted TEXT,  -- 加密存储
  refresh_token TEXT,
  expires_at INTEGER,
  is_active BOOLEAN
);
```

#### 5.2 社媒内容发布队列

```sql
CREATE TABLE social_posts (
  id TEXT PRIMARY KEY,
  account_id TEXT,
  content TEXT,
  media_urls JSON,
  scheduled_time INTEGER,
  status TEXT,                  -- 'pending', 'approved', 'published', 'failed'
  actual_post_id TEXT,
  engagement_stats JSON,        -- 点赞、评论等
  created_at INTEGER
);
```

#### 5.3 网站运营任务日志

```sql
CREATE TABLE site_ops_logs (
  id TEXT PRIMARY KEY,
  agent_name TEXT,
  action TEXT,                  -- 'seo_check', 'generate_blog', 'analyze_traffic'
  result_summary TEXT,
  details JSON,
  created_at INTEGER
);
```

### 6. 与 Nuwax 的集成

#### 6.1 模板发布
- 在 Nuwax 市场创建“网站运营 AI Team”套件，包含所有 Agent 定义和默认配置。
- 用户安装时，ProClaw 自动下载并注册这些 Agent，并引导用户完成首次配置（绑定网站分析账户、社媒账号等）。

#### 6.2 跨用户共享
- 支持用户将自己优化后的运营 Team 配置（如更有效的内容 prompt）导出为“自定义模板”，上传到 Nuwax 供其他用户使用（可选付费）。

#### 6.3 区域市场包
- Nuwax 提供“欧美社媒包”、“东南亚社媒包”、“国内社媒包”作为附加组件，用户可按需安装。

### 8. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 社媒 API 限制（频率、权限） | 使用官方 SDK 或第三方聚合服务；实现请求重试和降级策略。 |
| 内容质量不佳（尤其小语种） | 在内容生成后增加人工审核（确认卡片），并收集反馈微调 prompt。 |
| 合规风险（不同区域内容法规） | 内置内容过滤器（敏感词库），支持 Boss 设置黑白名单。 |
| 成本控制（大模型调用次数） | 限制每日生成次数，合并任务；本地模型与云端混合使用。 |
