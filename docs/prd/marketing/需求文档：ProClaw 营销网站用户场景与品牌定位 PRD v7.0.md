### 1. 背景与目标

#### 1.1 背景
- 营销网站当前版本内容偏向"功能陈述"，用户难以快速理解 ProClaw 对自己有什么具体价值。
- 目标用户是**不懂技术的小商家、个体户、创业学生**——他们有想法、有时间，但完全不具备部署 AI 或 Agent 的能力。他们需要的不是功能列表，而是"我能用它解决什么问题"。
- 产品已分化为两个清晰版本：**ProClaw Plus（进销存版）** 和 **虚拟公司版**，各自的适用场景不同，需要在营销网站上明确传达。

#### 1.2 目标
- 重新定义品牌核心定位，从"功能驱动"转向"场景驱动 + 用户收益驱动"。
- 设计一套覆盖主要目标人群的**真实使用场景**，用于营销网站的 Use Cases 页面重构。
- 建立统一的文案语调和叙事框架，确保全站传达一致。

---

### 2. 品牌核心定位

#### 2.1 一句话定位

> **ProClaw 是普通商户用上 AI Agent 团队的成本最低、路径最短的方式。**

#### 2.2 核心叙事

```
不懂技术 → 不需要部署 → 下载安装 → 跟它说话 → 它帮你干活。
```

关键洞察：
- 目标用户连"Agent"这个词都没听过，但他们知道"我忙不过来"。
- **卖的不是 AI 技术，卖的是"省心"**。
- 竞品在比功能多少，ProClaw 比的是"用户什么都不用懂"。

#### 2.3 品牌语调

| 维度 | 原则 |
|------|------|
| **语言风格** | 口语化、接地气，像朋友推荐一个好用的东西 |
| **避免词汇** | "部署""配置""工作流""上下文协议""集成"等技术人员用语 |
| **替换策略** | "部署"→"安装"，"配置"→"设置"（或用具体操作代替），"工作流"→"自动帮你做" |
| **情感调性** | 理解用户的忙和累，但不贩卖焦虑；传递"这事儿其实没那么复杂"的轻松感 |

### 3. 产品版本划分

#### 3.1 ProClaw Plus（进销存版）

**一句话描述**：给实体商户的 AI 经营工具，管商品、管库存、管订单，还能一键开网店。

**适用用户**：
- 个体户 / 夫妻店（便利店、小超市、五金店、服装店）
- 外贸柜台小老板
- 外贸批发商
- 小型工厂 / 作坊

**核心卖点**：
- 聊天就能管库存、查利润
- 一键开通云商城，商品自动同步
- 数据在自己电脑里，安全放心

#### 3.2 虚拟公司版

**一句话描述**：零成本搭建你的虚拟公司，AI Agent 当员工，帮你管钱管事管客户。

**适用用户**：
- 职业学校学生（接近零成本创业实践）
- 物业 / 市场管理方
- 社区团长 / 微商
- 自由职业者 / 小团队

**核心卖点**：
- 不需要注册公司，AI 帮你搭框架
- 财务 Agent 自动记账、出报表
- 可以自己装更多 Agent 扩展功能

### 4. 用户场景详述

#### 4.1 场景一：职业学校学生 — 零成本创业实践

| 项目 | 内容 |
|------|------|
| **角色画像** | 职业学校的电商/管理专业学生，有创业想法但没本钱，学校要求交创业方案但不知道从哪开始 |
| **当前痛苦** | 想试电商但没有商品、不懂建站、注册公司太贵太麻烦、同学协作全靠微信群消息乱飞 |
| **用 ProClaw 之后** | 下载虚拟公司版 → CEO Agent 引导设定"公司"方向和目标 → 用财务 Agent 做模拟预算 → 同学分工用任务 Agent 分配工作 → 期末导出完整创业方案和模拟经营数据 |

**推荐版本**：虚拟公司版

**关键叙事**：
> "不用注册公司、不用投一分钱。下载 ProClaw 虚拟公司版，你就是 CEO，AI 是你的员工。课堂上学的理论，这里直接实操。"

#### 4.2 场景二：个体户 / 夫妻店 — 从手工记账到 AI 管店

| 项目 | 内容 |
|------|------|
| **角色画像** | 社区便利店/小超市老板，开店 3-5 年，所有账本用手写或 Excel，库存靠"眼睛看+脑子记"，想开网店但觉得太复杂 |
| **当前痛苦** | 记账乱、月底算不清赚了多少亏了多少、进货全凭经验经常压货或缺货、客户问有没有网店只能说"没有" |
| **用 ProClaw 之后** | 下载 ProClaw Plus → 花半小时把商品录进去 → 每天卖货时顺便记一笔 → 月底对 AI 说"帮我算算上个月利润" → 发现有网店功能，点一下开通，商品自动上架 → 第二天就有老客户在网店下单了 |

**推荐版本**：ProClaw Plus（进销存版）

**关键叙事**：
> "不用学 Excel，不用请会计。跟它聊天就能把账算清楚。想开网店？跟它说一声，店就建好了。"

#### 4.3 场景三：外贸柜台小老板 — 多品类精细化管控

| 项目 | 内容 |
|------|------|
| **角色画像** | 在批发市场租柜台的电子产品/配件老板，SKU 几百个，颜色/规格/型号组合多，客户来自全国各地甚至海外 |

**推荐版本**：ProClaw Plus（进销存版）

**关键叙事**：
> "几百个 SKU？不用怕。ProClaw 的 SPU-SKU 模式专门处理多规格商品，颜色、尺寸、批次自动归类。客户欠了多少钱，AI 比你记得清楚。"

**关键叙事**：

### 5. 营销网站 Use Cases 页面策划

#### 5.1 页面结构

```
/use-cases（应用场景页）
│
├── 页面标题区
│   └── 主标题："选对你的场景，看看 ProClaw 能帮什么忙"
│
├── 版本切换 Tab
│   ├── ProClaw Plus（进销存版）→ 显示场景 2/3/4/7
│   └── 虚拟公司版 → 显示场景 1/5/6
│   └── [默认] 全部显示
│
├── 场景卡片列表（每个场景一张卡片）
│   ├── 左侧：角色插画/图标
│   ├── 右侧上：一段故事（角色 → 痛苦 → 用了之后）2-3 句话
│   ├── 右侧下：推荐版本标签 + "了解更多"链接
│   └── 共计 7 个场景
│
└── 底部 CTA："不确定选哪个版本？下载试试，两个版本都免费"
```

#### 5.2 设计原则

- **每个场景卡片必须让目标用户 3 秒内"看到自己"**——用角色名称而非抽象标签
- **用故事而非功能列表**——"以前……现在……"的对比结构
- **弱化技术词汇，强化结果**——不说"SPU-SKU 模式"，说"颜色规格再多也不乱"
- **每个场景标注推荐版本**，降低选择焦虑

#### 5.3 与现有 Features 页面的关系

| 页面 | 目的 | 内容侧重 |
|------|------|---------|
| **/features**（功能全景） | 让已经感兴趣的用户深入了解能力 | 按模块展示功能，适合技术导向用户 |
| **/use-cases**（应用场景） | 让新访客快速对号入座 | 按角色讲故事，适合不懂技术的用户 |

两个页面从不同角度服务不同类型的访客，不重复。
