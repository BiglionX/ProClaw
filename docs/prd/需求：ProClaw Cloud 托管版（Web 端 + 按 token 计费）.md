“个体户连电脑都没有，直接在 proclaw.cc 网页上登录使用，数据存在云端，按 token 消耗收费”的模式，**不是传统 SaaS（软件即服务，通常按订阅制收费）**，而更接近：

## 建议命名：**ProClaw Cloud 托管版** 或 **ProClaw Cloud 按量版**

具体可细分为：

| 名称 | 说明 |
|------|------|
| **ProClaw Cloud 托管版** | 强调用户无需自己部署，服务完全由 ProClaw 官方在云端托管运行。 |
| **ProClaw Cloud 弹性版** | 强调资源弹性伸缩，按实际使用的 token 付费，不绑定固定套餐。 |
| **ProClaw Cloud 无服务器版** | 借用 Serverless 概念，用户只关心业务，不关心服务器。 |
| **ProClaw Web 版** | 最直白：通过浏览器访问，无需安装任何桌面端或 App。 |

> 从商业模式看，这种 **按 token 消耗计费、用户不持有服务器** 的模式，本质上是 **“托管式按量付费云服务”**（Managed Pay-as-you-go Cloud Service）。在行业里，这类服务常被归为 **“公有云版”** 或 **“云原生版”**。

## 完整需求：ProClaw Cloud版（Web 端 + 按 token 计费）

### 1. 背景与目标

#### 1.1 背景
- 现有 ProClaw 需要用户在本地电脑安装桌面端，但许多个体户、小微创业者甚至没有电脑（仅有手机/平板），或不愿管理本地服务器。
- 同时，ProClaw 已设计按 token 消耗计费的 Pro 云备份版，但该版本仍依赖本地桌面端作为主服务器。
- 为覆盖“零硬件”用户群体，需提供**纯云端托管版本**：用户通过浏览器访问 `app.proclaw.cc` 即可使用所有功能，数据存储在 ProClaw 官方云数据库，按 token 用量付费。

#### 1.2 目标
- 推出 **ProClaw Cloud 托管版**，为没有本地设备的用户提供完整服务。
- 保留原有桌面端 + 移动端直连架构，但新增云端多租户架构。
- 计费模式：按 token 消耗（与 Pro 云备份版保持一致），无固定月费或年费（可选小额存储费）。
- 数据隔离：每个用户独立数据库（或 schema），加密存储。

---

### 2. 用户故事

- **作为** 个体户店主，我没有电脑，只有一部手机。我直接在浏览器打开 `app.proclaw.cc` 注册账号，支付少量 token 预充值，就可以使用财务管理 Agent、生成订单、与客户聊天。所有数据都存在云端，我不用担心备份。
- **作为** 自由设计师，我偶尔需要用到 ProClaw 的 AI 识别订单功能，我不想安装任何软件。我按需充值 token，用完即止。

### 3. 技术架构

#### 3.1 架构图

```
用户浏览器
    │
    │ HTTPS
    ▼
┌─────────────────────────────────────────────────┐
│         ProClaw Cloud 托管版 (proclaw.cc)        │
│  ┌─────────────────────────────────────────────┐ │
│  │  前端：Next.js (React) + Tailwind           │ │
│  │  - 响应式设计（桌面/平板/手机）              │ │
│  │  - 与桌面端功能对等（聊天、进销存、Agent）   │ │
│  └─────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  后端：Node.js (NestJS) 或 Rust (axum)       │ │
│  │  - 多租户管理                                │ │
│  │  - WebSocket 服务（聊天、实时通知）          │ │
│  │  - 进销存、AI、Agent 逻辑                    │ │
│  └─────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  数据库：PostgreSQL (隔离: schema per tenant)│ │
│  │  缓存：Redis                                 │ │
│  │  对象存储：S3（文件/图片）                   │ │
│  └─────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  计费服务：按 token 消耗实时扣费             │ │
│  └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

#### 3.2 与桌面版的关系
- **代码复用**：前端 React 组件可直接复用（适配响应式），后端业务逻辑（Rust 库）可编译为 WebAssembly 或通过 Node.js 绑定调用。
- **数据不互通**：云端版与桌面版是独立账户体系，用户可选择导出/导入数据。
- **未来打通**：提供“一键从云端版导出并迁移到桌面版”功能。

### 4. 功能需求（与桌面版对等）

由于是纯 Web 端，需实现全部核心功能：
- 进销存（商品、采购、销售、库存、报表）
- 聊天（文字、图片、文件、语音消息）
- AI 订单识别（拍照/上传图片）
- Agent 框架（财务管理、CEO Agent、市场 Agent 等）
- 设备授权（不需要，因为无本地设备）
- 文件存储（云端 S3）
- WebSocket 推送通知

**差异点**：
- 无“局域网直连”模式，所有通信走 HTTPS/WSS。
- 支持离线模式：通过 Service Worker 缓存部分静态资源，但业务数据需在线。
- 移动端响应式布局，代替独立 App（也可发布为 PWA）。

### 5. 计费模式（按 token 消耗）

与 Pro 云备份版 token 定价一致（参考 PRD v4.0/v5.0 附录）：

| 消耗项 | token 消耗 |
|--------|------------|
| 业务 API 请求（增删改查） | 1 token |
| 聊天消息（发送/接收） | 1 token |
| WebSocket 长连接（1 小时） | 5 token |
| 文件上传（每 1 MB） | 10 token |
| AI 识别图片（每次） | 5 token |
| 存储空间（每 GB·月） | 100 token |

用户需先充值 token（1 元 = 1000 token），系统实时扣费。余额不足时限制部分操作。

**免费额度**：新用户赠送 1000 token（用于体验），无有效期。

---

### 6. 数据安全与隐私

- 数据库字段加密：用户敏感数据（如手机号、地址）使用 AES-256 加密。
- 多租户隔离：使用 PostgreSQL 的 schema per tenant，防止数据泄漏。
- 备份：每日自动备份，保留 30 天。
- 用户可随时导出所有数据（JSON/CSV）并彻底删除账户。

### 8. 风险与应对

| 风险 | 应对 |
|------|------|
| 多人同时在线导致服务器压力 | 使用云原生自动扩容（K8s + HPA）；按 token 收费覆盖成本。 |
| 用户流失后数据残留 | 提供自动删除策略（账户停用 90 天后删除）。 |
| Web 端性能不如桌面端 | 针对低端设备优化，使用虚拟滚动、懒加载。 |
