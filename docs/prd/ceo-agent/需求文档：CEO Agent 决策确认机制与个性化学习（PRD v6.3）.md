### 1. 背景与目标

#### 1.1 背景
- PRD v6.2 引入了 CEO Agent 作为主控官，能够根据 Boss 的宏观意图更新项目上下文协议（PCP）并分派任务。
- 大模型存在“幻觉”风险，可能误解 Boss 意图或生成不合理任务，因此引入了确认卡片机制。
- Boss 的核心价值在于**决策**。不同 Boss 的决策风格、业务重点和成功路径各不相同，同一套 AI Agent 组合在不同用户手中会产生截然不同的公司发展轨迹。

#### 1.2 目标
- **强确认机制**：所有重要决策（创建/修改 PCP 条目、分派任务、子 Agent 关键操作）均需生成确认卡片，Boss 可点击“确认”或“拒绝”。拒绝时可补充原因或修改。
- **个性化学习**：CEO Agent 通过记录 Boss 的每一次确认/拒绝及其反馈，逐渐拟合 Boss 的决策偏好，减少未来需要确认的频率，同时提高决策匹配度。
- **可解释性**：CEO Agent 在生成决策时，附带**决策理由**（基于 PCP 中的哪些条目、历史数据等），让 Boss 更容易理解 AI 的思考过程。
- **发展轨迹差异化**：允许 Boss 导出/分享自己的“虚拟公司发展配置包”（含 PCP 历史、决策偏好模型、Agent 组合），形成独特的“公司灵魂”。

### 2. 决策确认卡片设计

#### 2.1 触发场景
- **创建或修改 PCP 条目**（如新增目标、改变约束条件）。
- **分派新任务给子 Agent**（尤其是跨 Agent 或高优先级任务）。
- **子 Agent 提出需要 Boss 批准的请求**（如财务 Agent 的预算调整、营销 Agent 的对外承诺）。
- **CEO Agent 主动提议撤销或暂停某个进行中的目标/任务**。

#### 2.2 卡片内容
- **标题**：清晰说明决策类型（例如“新增项目目标”、“分派营销任务”）。
- **详细描述**：具体内容，如目标描述、任务说明、预期结果。
- **决策依据**：引用相关 PCP 条目、历史数据或外部条件（例如“根据 PCP 中‘Q3 海外市场目标’自动派生”）。
- **可选选项**：
  - ✅ **确认**（接受并执行）
  - ❌ **拒绝**（拒绝，并弹出输入框让 Boss 填写原因或给出修改建议）
  - ✏️ **编辑**（允许 Boss 直接修改卡片中的内容，然后确认）
  - ⏰ **稍后提醒**（推迟决策，设置具体时间）
- **附加信息**：该决策的预估影响（如预算占用、人员工时、风险等级）。

#### 2.3 对话式交互
- 确认卡片出现在 CEO Agent 的聊天窗口中，作为一条富文本消息。
- Boss 点击按钮后，CEO Agent 回应：“已确认，任务已派发给财务 Agent。”或“已拒绝，原因已记录，我会调整方案。”
- 支持批量确认（当有多个类似决策时，可生成汇总卡片）。

### 3. 个性化学习机制

#### 3.1 数据收集
在 SQLite 中增加 **决策日志表**：

```sql
CREATE TABLE ceo_decision_logs (
  id TEXT PRIMARY KEY,
  decision_type TEXT,          -- 'context_add', 'task_dispatch', 'agent_request', etc.
  proposed_content JSON,       -- 原始建议内容
  boss_decision TEXT,          -- 'approved', 'rejected', 'edited'
  boss_feedback TEXT,          -- 用户填写的反馈文字
  final_content JSON,          -- 最终生效的内容（如被编辑后）
  context_snapshot JSON,       -- 当时的 PCP 快照
  created_at INTEGER,
  approved_at INTEGER
);
```

#### 3.2 偏好模型
- **短期偏好**：基于最近 50 次决策，计算 Boss 对各类建议的接受率、常见修改方向（例如更偏好保守预算、喜欢详细数据支撑）。
- **长期拟合**：使用轻量级机器学习（如逻辑回归）或 LLM 微调（LoRA）来预测 Boss 的决策倾向。初期可简化：利用 LLM 在 prompt 中注入历史决策摘要：“根据你过去的决策，你通常拒绝超过 5000 元的未预审预算，请按此原则生成建议。”
- **主动询问**：当 CEO Agent 不确定时，可以主动问：“老板，关于这个问题，您希望我更倾向于成本控制还是快速推进？” 根据回答更新偏好权重。

#### 3.3 拟合度指标
- 定义 **决策接受率** = 被确认的决策数 / 总决策建议数。
- 随着公司运行时间增长，接受率应逐步上升（目标 > 85%）。
- 若接受率突然下降，CEO Agent 可主动询问：“最近我的建议似乎不太符合您的期望，能告诉我哪些方面需要改进吗？”

### 4. 差异化发展支持

#### 4.1 公司配置包导出/导入
- Boss 可以在设置中选择 **“导出公司发展配置包”**，包含：
  - PCP 所有条目（脱敏后可选）
  - 决策日志摘要（不包含敏感财务数据）
  - CEO Agent 的偏好模型参数（如权重向量）
  - 已安装的 Agent 列表及版本
- 导出的文件为加密 JSON，可分享给其他用户。
- 其他用户导入后，可获得类似的 CEO Agent 决策风格和项目上下文，形成 **“虚拟公司克隆”** 或 **“模板公司”**。

#### 4.2 公司发展时间轴
- 在项目仪表板中增加 **“公司时间轴”** 视图，以故事化方式展示虚拟公司的关键决策节点、里程碑、成功/失败案例。
- Boss 可以标注“高光时刻”或“转折点”，这些标注将作为个性化学习的正向/负向反馈。

#### 4.3 社区分享
- 可选：将脱敏后的公司配置包上传到 `nvwa.proclaw.cc` 的“公司案例库”，其他用户可浏览、评分、一键采用（需自行调整具体数据）。

### 5. 用户界面补充

#### 5.1 确认卡片样式
- 使用 MUI 的 `Card` 组件，背景色为浅黄（提示色），按钮使用醒目的颜色。
- 支持键盘快捷键：`Y` 确认，`N` 拒绝，`E` 编辑。

#### 5.2 决策历史视图
- 在 CEO Agent 聊天窗口侧边栏增加“决策历史”标签页，列出所有重要决策及其 Boss 反应。
- 支持按决策类型、时间范围筛选。

#### 5.3 偏好设置面板
- 在设置中增加“CEO Agent 偏好”，Boss 可以手动调整偏好权重（例如“预算敏感度”、“风险偏好”滑块）。

---

### 6. 对现有 API 的补充

- `proclaw.context.addWithApproval(entry)`：生成确认卡片，等待 Boss 反馈后才真正写入 PCP。
- `proclaw.tasks.dispatchWithApproval(task)`：同上。
- `proclaw.learning.getStats()`：返回决策接受率、最常拒绝的类型等统计。
- `proclaw.learning.updatePreference(field, value)`：手动更新偏好。

### 8. 风险与备选

- **过度依赖历史**：若 Boss 改变管理风格（如从保守转为激进），CEO Agent 可能滞后。应对：提供“重置偏好”按钮，或允许 Boss 手动切换“风格模板”。
- **决策疲劳**：确认卡片过多可能导致 Boss 厌烦。策略：合并同类决策、提高拟合度后自动跳过低风险决策（Boss 可设置“自动确认阈值”）。
- **隐私问题**：导出配置包可能泄露商业思路。应对：默认强制脱敏（替换真实名称为占位符），且仅限用户主动分享。
