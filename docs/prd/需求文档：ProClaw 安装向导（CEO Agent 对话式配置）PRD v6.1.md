### 1. 背景与目标

#### 1.1 背景
- ProClaw（PRD v6.0）已定义 Agent 化架构，用户即 Boss，预装财务管理 Agent，支持从市场安装其他 Agent。
- 安装过程是用户对产品的第一印象，传统“下一步”式向导枯燥且缺乏个性。
- 借鉴“AI 原生”理念，让 **CEO Agent** 在安装过程中以对话形式引导 Boss 完成配置，增强趣味性和代入感，同时收集关键信息（路径、公司名、大模型偏好等）。

#### 1.2 目标
- 设计一套 **CEO Agent 主导的安装向导**，全程自然语言交互，用户只需回答几个简单问题即可完成安装。
- CEO Agent 需具备：询问安装路径、检查磁盘空间并建议、收集公司名称、引导大模型配置（本地模型路径或 Pro Cloud 模型选择）。
- 安装结束后，CEO Agent 成为虚拟公司的“默认助手”，可在聊天窗口中随时呼唤。

---

### 2. 用户故事

- **作为** 首次安装 ProClaw 的用户（Boss），我打开安装包后，看到一个拟人化的“CEO Agent”头像，它用礼貌的语气问我：“老板，您希望把我们的虚拟公司安装在哪个位置？”我选择一个文件夹，它检查空间后说：“这个位置有 50GB 可用，足够使用。很好！”然后问我公司名字，我输入“云创科技”，它说：“好名字！公司已注册。”接着问我大模型配置，我选择“暂时使用 Pro Cloud 提供的模型”，安装完成。整个过程感觉很贴心。
- **作为** 技术型用户，我在大模型配置步骤选择“本地部署”，指定 Ollama 或 llama.cpp 路径，CEO Agent 自动测试连接并提示成功。

### 3. 安装流程对话设计

#### 3.1 启动安装
- 用户双击安装包（或首次启动桌面端），Tauri 窗口打开，显示 CEO Agent 头像（动画），并发出欢迎语（文字 + 语音可选）。
- **CEO Agent**: “您好，老板！我是您的虚拟公司 CEO Agent。很高兴为您效劳。让我们一起把公司建立起来吧。”

#### 3.2 步骤 1：选择安装位置
- **CEO Agent**: “老板，您希望把我们的虚拟公司安装在哪个位置？请选择一个文件夹，我会为您检查可用空间。”
- 用户操作：弹出系统文件夹选择对话框，或输入路径。
- **CEO Agent**（自动检查）：
  - 如果空间 < 1GB → “这个位置空间有些紧张（XX MB 可用）。建议至少保留 1GB 空闲。换个地方吧？”
  - 如果空间 >= 1GB → “很好，这个位置有 XX GB 可用，足够公司发展。我们就落户在这里。”

#### 3.3 步骤 2：公司命名
- **CEO Agent**: “老板，已经我们的虚拟公司已经落户好了，请给我们公司取个名字吧（可以是真实公司名或任何您喜欢的名字）。”
- 用户输入公司名称（长度限制 1-50 字符，自动过滤非法字符）。
- **CEO Agent**（回应）：“『{公司名}』，寓意深远，是个好名字！公司注册成功。”

#### 3.4 步骤 3：大模型配置
- **CEO Agent**: “老板，为了增加公司的能力，下面需要部署大模型。如果现在不设置，也可以先直接使用 Pro Cloud 提供的自选大模型，以后再设置也可以。请选择：”
- 提供两个选项按钮：
  - **“使用 Pro Cloud 模型（推荐，无需配置）”**
  - **“本地部署模型（需要已有 Ollama/llama.cpp）”**
- 根据用户选择：
  - **Pro Cloud**：记录配置为 `model_provider = 'procloud'`，后续 AI 能力默认调用云端 API（需 Pro 版订阅或按量付费）。
  - **本地部署**：弹出路径选择，要求用户指定可执行文件或模型目录。CEO Agent 测试连接（例如调用 `ollama list` 或发送测试 prompt）。成功则提示“大模型已就绪”，失败则提示“连接失败，请检查路径后重试”并回退到选择界面。

#### 3.5 步骤 4：安装完成 & CEO Agent 常驻
- **CEO Agent**: “老板，一切就绪！虚拟公司已启动。您可以在聊天窗口随时找到我。现在，让我们开始工作吧！”（显示“进入工作区”按钮）
- 点击后关闭向导，进入主界面。
- 在聊天联系人列表中，自动添加 **“CEO Agent”** 为好友，并打开默认会话窗口，显示欢迎消息和快捷操作（例如“创建任务”、“查看财务”）。

### 4. 技术实现要点

#### 4.1 CEO Agent 的实现
- CEO Agent 本身是一个 **内置特权 Agent**，不由市场安装，其逻辑硬编码在桌面端。
- 它使用相同的 Agent API 桥接（可读写用户配置、调用系统功能）。
- 界面：在安装向导中嵌入对话气泡，使用 React 组件实现打字机效果。

#### 4.2 配置存储
- 收集的信息存储于桌面端 SQLite 的 `system_config` 表中：
```sql
CREATE TABLE system_config (
  key TEXT PRIMARY KEY,
  value TEXT
);
-- 示例：
-- ('install_path', '/home/user/proclaw_data')
-- ('company_name', '云创科技')
-- ('model_provider', 'procloud') 或 ('local', '/usr/local/bin/ollama')
```

#### 4.3 安装路径处理
- 实际数据存储（SQLite、文件上传）与程序安装路径分离。用户选择的路径用于存放数据目录。
- Tauri 后端需具有读写该路径的权限，并在 `tauri.conf.json` 中声明 `"fs:allow-write"` 等。

#### 4.4 大模型连接测试（本地模式）
- 若用户选择本地 Ollama，则调用 `ollama list` 命令获取模型列表，或通过 HTTP API（`http://localhost:11434/api/tags`）检查服务是否运行。
- 若用户选择 llama.cpp server，则尝试调用 `/v1/models` 端点。

#### 4.5 安装后首次启动
- 如果系统检测到已配置（`system_config` 中有 `install_path`），则跳过向导，直接进入主界面。
- 允许用户在设置中重新运行向导（“重新配置公司”），但需注意不要覆盖已有数据（除公司名外）。

---

### 5. 界面设计要点

- 向导窗口无边框或定制标题栏，背景使用渐变或虚拟公司风格的图片。
- CEO Agent 头像使用 SVG 动画（眨眼、微笑）。
- 输入区域：文本输入框（用于公司名）+ 按钮（用于路径选择和模型选项）。
- 步骤指示器：简单的小圆点，显示当前在第几步。
- 支持键盘快捷键：Enter 确认、ESC 取消（但不要轻易取消）。

---

### 6. 与现有 PRD 的整合

- 本向导为 **ProClaw** 的入口，替代传统安装程序界面。
- 安装完成后，自动初始化：
  - 财务管理 Agent（内置）。
  - CEO Agent 常驻聊天列表。
  - 根据大模型配置，后续 AI 功能（如智能客服、自动报表）使用对应模型。
- 用户首次进入主界面时，可引导其打开 Agent 市场安装更多 Agent。

---

### 7. 实施步骤

1. **桌面端安装包改造**：Tauri 应用首次运行时检测是否已配置，若否，启动向导窗口。
2. **实现向导组件**：React 组件，包含多步对话逻辑。
3. **实现路径选择与空间检查**：调用 Tauri 的 `dialog` 和 `fs` API。
4. **实现公司名验证与存储**。
5. **实现大模型配置测试**：本地模式调用 subprocess 或 HTTP 请求。
6. **将配置写入 SQLite**。
7. **安装后初始化默认 Agent**：注册 CEO Agent 为内置用户。
8. **测试各种路径**（无权限、空间不足、Ollama 未运行等）。

---

### 8. 风险与备选

- **空间检查不准**：不同操作系统对剩余空间的计算可能有差异，使用 `fs::metadata` 和 `statvfs` 结合。
- **Ollama 未安装**：提示用户先安装 Ollama 或改用 Pro Cloud。
- **用户跳过公司名**：提供默认名“我的虚拟公司”。
- **安装中途退出**：允许用户保存已填信息，下次继续。
