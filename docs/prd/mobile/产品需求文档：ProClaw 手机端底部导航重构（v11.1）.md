## 1. 背景与动机

### 1.1 现状问题

当前底部导航项：
```
首页 | Agent | 商品 | 供应链 | 我的（5项）
```

- 最多 5 个 Tab，挤占有限的底部导航空间
- 供应链作为独立 Tab 却包含子页面，入口层级混乱
- "商品"与"供应链"两个 Tab 对手机用户而言过于专业化，认知门槛高
- 角色动态隐藏 Tab 导致底部栏数量不固定，违反 HIG 规范
- AI Agent、AI Team 隐藏在独立的 Agent Tab 中，与用户工作流割裂

### 1.2 设计理念

> **手机不是桌面端的缩小版，而是一个"人对人"的通讯式工作台。**

借鉴 QQ/微信的通讯录设计哲学：
- **联系人**是一切业务的起点（客户、同事、AI 助手、AI 团队）
- **消息**是工作流的自然流转通道
- **我的**是个人工作台，看板 + 快捷操作 + 设置统一管理

## 2. 底部导航方案

### 2.1 Tab 定义

| 序号 | Tab 名称 | 图标（MaterialCommunityIcons） | 核心功能 |
|------|---------|-----------------------------|---------|
| 1 | **联系人** | `account-multiple` | 个人联系人 + AI Agent + AI 团队，统一入口 |
| 2 | **消息** | `chat-processing` | 聊天会话列表，与联系人/Agent 的通信记录 |
| 3 | **我的** | `account-circle` | 数据看板 + 快捷操作 + 设置 + 身份管理 |

### 2.2 设计原则

1. **固定导航**：3 个 Tab 固定显示，不对任何角色隐藏
2. **始终可见**：无需登录/选中身份即可看到导航，未登录时提醒引导
3. **图标 + 文字**：底部显示图标 + 标签，选中态使用品牌色 `#6366f1`
4. **Tab Bar 样式**：
   - 背景色：白色
   - 选中激活色：`#6366f1`
   - 未选中色：`#999`
   - 高度：60px
   - 顶部 1px 分割线：`#eee`

## 3. 联系人 Tab（ ContactsTab ）详情

### 3.1 页面结构

采用 **分组通讯录** 布局，分为三个区：

```
┌─────────────────────────────┐
│ 🔍 搜索联系人...              │  ← 搜索栏
├─────────────────────────────┤
│ ▼ 个人联系人                   │  ← 可折叠分组
│   👤 张伟   客户 · 恒达商贸    │
│   👤 陈志远 同事 · 技术部      │
│   👤 周建国 供应商 · 鼎丰电子  │
│   ...                        │
├─────────────────────────────┤
│ ▼ AI Agent                   │  ← 可折叠分组，不可删除
│   🤖 小如（秘书）              │  ← 固定，不可卸载
│   🤖 CEO Agent               │  ← 固定，不可卸载
│   🤖 客服 Agent              │  ← 固定，不可卸载
├─────────────────────────────┤
│ ▼ AI Team                    │  ← 可折叠分组
│   🧠 AI运营团队               │  ← 已激活 AI 团队
│   🧠 自媒体运营团队            │
│   🧠 更多...                  │
│   📥 从市场安装更多 AI 团队    │  ← 跳转插件市场/AI市场
└─────────────────────────────┘
```

### 3.2 个人联系人

**数据来源**：复用现有 `ContactsScreen` 的业务联系人数据（客户/供应商/同事）

- 联系人类型：`customer`（客户）、`supplier`（供应商）、`colleague`（同事）、`friend`（朋友）
- 类型视觉区分：
  - 客户→ `account-star` 图标/绿色
  - 供应商→ `truck-delivery` 图标/橙色
  - 同事→ `account-tie` 图标/紫色
  - 朋友→ `account-heart` 图标/粉色（新增）
- 右侧留白 → 点击弹出操作菜单（电话/消息/编辑/删除）
- 长按 → 批量多选、转发分享

### 3.3 AI Agent

**定义**：ProClaw 系统自带的 AI 助手，固定存在，不可删除

| Agent | 名称 | 图标 | 默认状态 | 说明 |
|-------|------|------|---------|------|
| 秘书 | 小如 | `robot-happy` | ✅ 已启用 | 个人工作秘书，处理日程/提醒/查询 |
| CEO | CEO Agent | `account-tie-hat` | ✅ 已启用 | 经营决策分析，数据洞察 |
| 客服 | AI 客服 | `headset` | ⚠️ 需配置 | 客户服务响应，需绑定 NvwaX API |

交互行为：
- 点击 → 跳转到与该 Agent 的**消息会话**（打开 消息 Tab 的对应会话）
- 滑动/长按 → 查看 Agent 详情、启用/禁用开关、配置
- 状态标识：绿色圆点（在线）、灰色圆点（离线/未配置）

### 3.4 AI Team

**定义**：用户从 AI 市场/插件商店安装的行业 AI 团队，可动态增加/减少

- 每个 AI 团队显示名称 + 简要描述 + 成员数量
- 点击 → 跳转到团队会话（消息 Tab 的群聊）
- 底部「从市场安装更多 AI 团队」→ 跳转插件市场，筛选 AI 团队类插件
- 空状态时显示引导：`暂无 AI 团队，去市场看看吧`

### 3.5 搜索功能

搜索范围覆盖全部三类联系人：
- 个人联系人：按姓名/电话/公司/部门搜索
- AI Agent：按名称搜索
- AI Team：按团队名称搜索

## 4. 消息 Tab（ MessagesTab ）详情

### 4.1 页面布局

```
┌─────────────────────────────┐
│ 🔍 搜索会话...               │  ← 搜索栏
├─────────────────────────────┤
│ 📌 置顶会话                   │  ← 用户手动置顶
│   ┌───────────────────────┐ │
│   │ 🤖 小如  14:20 来新消息了│ │  ← Agent 消息
│   └───────────────────────┘ │
├─────────────────────────────┤
│ 💬 最近会话                   │  ← 按时间倒序
│   ┌───────────────────────┐ │
│   │ 👤 张伟  13:50 确认报价  │ │  ← 联系人消息
│   ├───────────────────────┤ │
│   │ 🧠 AI运营团队 昨 周报   │ │  ← AI 团队群聊
│   ├───────────────────────┤ │
│   │ 👥 采购群聊 昨 采购清单  │ │  ← 群组消息
│   └───────────────────────┘ │
└─────────────────────────────┘
```

### 4.2 会话类型

| 类型 | 来源 | 图标 | 交互 |
|------|------|------|------|
| 个人消息 | 与业务联系人的对话 | 联系人头像 | 点击进入 1v1 聊天 |
| Agent 消息 | 与 AI Agent 的对话 | Agent 自定义图标 | 点击进入 AI 对话界面 |
| AI 团队群聊 | 与 AI 团队的多方对话 | 团队图标 | 点击进入群聊界面 |
| 群组 | 多人业务讨论组 | 群头像 | 点击进入群聊 |

### 4.3 交互功能

- **左滑** → 标记已读/未读
- **右滑** → 置顶/取消置顶
- **长按** → 批量操作（删除、标记已读、转发）
- **下拉** → 刷新会话列表
- **空状态**：`暂无消息，开始与联系人对话吧`
- **新消息提醒**：Tab 右上角显示未读数角标（badge）

### 4.4 消息列表项展示

每项显示：
- 左侧头像（联系人头像 / Agent 图标 / 群组图标）
- 中间名称 + 最新消息预览（单行，超出省略）
- 右侧时间戳 + 未读数角标

## 6. 技术实现要点

### 6.1 导航结构调整

移除 5 个 Tab，改为 3 个固定 Tab（联系人、消息、我的）：
- 删除 `ROLE_TAB_ACCESS` 角色 → Tab 映射逻辑
- 删除 `getVisibleTabs` 函数及动态隐藏逻辑
- 已安装插件路由从底部 Tab 解耦，改为在「消息」和「我的」内页中嵌入

### 6.2 联系人 Tab 改造

**现有 `ContactsScreen.tsx` 改造**：
1. 保留现有联系人数据的加载逻辑
2. 在 FlatList 上方增加分组 header：个人联系人 / AI Agent / AI Team
3. AI Agent 数据来自 `agentRuntimeBridge.getInstalledAgents()`
4. AI Team 数据来自已安装插件中的 AI 团队类型插件
5. 搜索范围扩展到全部三类数据

### 6.3 消息 Tab 新建

**新建 `MessagesScreen.tsx`**：
1. 会话列表从本地数据库 `chat_sessions` 表读取（若无则显示空态）
2. 消息预览从 `chat_messages` 表读取最新一条
3. 未读数统计
4. 进入消息后导航到消息详情页

需要确认或新建的表结构：
```sql
CREATE TABLE IF NOT EXISTS chat_sessions (
  id TEXT PRIMARY KEY,
  session_type TEXT NOT NULL,       -- 'personal' | 'agent' | 'team' | 'group'
  target_id TEXT NOT NULL,          -- 联系人ID / Agent ID / 团队ID
  target_name TEXT NOT NULL,
  target_icon TEXT,
  last_message TEXT,
  last_message_time INTEGER,
  unread_count INTEGER DEFAULT 0,
  is_pinned INTEGER DEFAULT 0,
  created_at INTEGER DEFAULT (CAST(strftime('%s','now') AS INTEGER))
);
```

### 6.4 我的 Tab 改造

1. 将 `HomeScreen.tsx` 的数据看板移入 `ProfileScreen.tsx` 顶部
2. 新增快捷操作网格入口
3. 保持原有设置列表在其后
4. 身份卡片置于顶部

### 6.5 已废弃页面的处理

- `AgentsScreen.tsx` → 不再作为独立页面，其功能由联系人 Tab 的 AI Agent 分组替代
- `ProductsScreen.tsx` → 入口调整到"我的" → 快捷操作，或由内部导航承载
- `SupplyChainScreen.tsx` → 入口调整到"我的" → 快捷操作

## 7. 用户旅程示例

### 场景 1：老板查看经营概况
1. 打开 App → 底部「我的」Tab
2. 顶部数据看板快速看到商品数、客户数、订单数
3. 点击商品数 → 跳转到商品列表

### 场景 2：联系供应商询价
1. 底部「联系人」Tab
2. 搜索"周建国"或在供应商分组找到
3. 点击 → 跳转到"消息"Tab 与该供应商的会话
4. 输入/发送消息

### 场景 3：让秘书安排日程
1. 底部「联系人」Tab
2. 在 AI Agent 分组点击"小如"
3. 跳转到"消息"Tab 与小如的对话
4. 发送"帮我安排下周二的会议"

### 场景 4：查看 AI 运营团队工作
1. 底部「联系人」Tab
2. 在 AI Team 分组点击"AI运营团队"
3. 跳转到群聊会话，查看团队输出/报告

### 场景 5：数据备份
1. 底部「我的」Tab
2. 快捷操作中点击「云备份」
3. 查看/执行备份
