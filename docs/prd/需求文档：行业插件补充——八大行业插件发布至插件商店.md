# 需求文档：行业插件补充 —— 八大行业插件发布至插件商店

> 版本：v1.0 · 基于既有餐饮/美业/宠物三大行业插件，补充覆盖八大高频批发零售业态  
> 关联文档：`需求文档：行业插件功能实现（餐饮 美业 宠物 Cloud）.md`  
> 目标：在 FlowHub 插件商店完成 8 个新行业插件的 manifest 定义、建表迁移、路由注册、AI Agent 创建及商店上架

---

## 一、背景与产品线澄清

### 1.1 产品线 vs 行业插件

| 类型 | 名称 | 定位 | 是否插件 |
|------|------|------|----------|
| **产品线** | ProClaw Plus（进销存版） | 个体户、批发商、工厂等实体经营 | ❌ 产品分类 |
| **产品线** | ProClaw Light（虚拟公司版） | 学生创业、物业/市场管理方、社区团长 | ❌ 产品分类 |
| **产品线** | ProClaw Cloud（云托管版） | 按 Token 计费的 Web 端托管服务 | ❌ 产品分类 |
| **行业插件** | 餐饮、美业、宠物 | 垂直行业深度定制 | ✅ 插件 |
| **行业插件** | 便利店、酒水批发、手机配件批发、食材配送、汽车配件、五金、装修材料、社区团购 | 批发零售细分业态 | ✅ 插件（新增） |

### 1.2 插件商店当前状态

- 本地已存在行业插件 manifest（仅 `catering` / `beauty` / `pet` 三个）
- FlowHub 插件商店数据库（`industry_plugins` 表）目前**无已发布插件**，数据为空
- 所有插件 manifest、迁移 SQL、路由页面、AI Agent 均完成开发后，需通过 FlowHub 后台 INSERT `status = 'published'` 后方可在商店展示

## 三、公共约束

### 3.1 共享页面复用

以下页面为全版本通用，所有行业插件通过 manifest 的 `navigation.remove` 控制可见性：

| 页面 | 路径 | 说明 |
|------|------|------|
| AI 首页 | `/` | AI 对话主页 |
| AI 团队 | `/teams` 或 `/ai-teams` | 全版本通用 |
| AI 知识库 | `/ai-knowledge` | 行业知识库 |
| 销售管理 | `/sales` | 销售订单/出库 |
| 库存管理 | `/inventory` | 库存盘点/预警 |
| 数据看板 | `/analytics` | 经营分析 |
| 供应链 | `/supplychain` | 采购/供应商管理 |
| 商品库 | `/products` | 商品 SKU 管理 |

### 3.2 路由注册方式

所有插件路由统一在 `src/App.tsx` 中注册，不随插件动态增减。manifest 的 `navigation.add` / `navigation.remove` 仅控制侧边栏可见性。

### 3.3 插件商店数据模型

插件信息存储在 FlowHub Supabase 的 `industry_plugins` 表中：

```sql
INSERT INTO industry_plugins (id, name, version, status, manifest_json, icon, description, min_app_version)
VALUES ('convenience-store', '便利店行业版', '1.0.0', 'published', '{}', '🏪', '...', '>=0.1.0');
```

1. **商店可见**：FlowHub 插件商店（`https://flowhub.proclaw.cc`）中能看到 8 个新插件卡片，包含图标、名称、描述、版本号

6. **命令可用**：前端调用 Tauri 后端命令返回合理的模拟数据或状态
