## 需求文档：ProClaw 内置商务秘书 Agent（PRD v8.5）

### 1. 背景与目标

#### 1.1 背景
- 当前桌面端右下角的浮动 AI 聊天窗口定位为通用「ProClaw 经营智能体」，身份模糊、缺乏人格化特征，且职责边界不清晰。
- 用户需要的是一个**专属数据参谋**——她能读懂全系统数据、记住老板关注什么、主动呈报关键信息，但**不做决策**。决策权属于 CEO Agent（PRD v6.2）。
- 参考豆包等AI助手的头像个性化设计，用户期望能自定义秘书的**名称、头像**，形成情感连接。
- 语音交流是未来必然趋势，需在设计阶段预留接口和交互入口。

#### 1.2 产品定位：秘书 vs CEO Agent 的职责边界

```
┌─────────────────────────────────────────────────────────┐
│                     Boss（老板）                          │
│                                                         │
│   ┌──────────────┐              ┌──────────────────┐    │
│   │  商务秘书      │  情报上呈    │   CEO Agent      │    │
│   │  (Secretary)  │ ──────────→ │   (主控官)        │    │
│   │              │ ←────────── │                  │    │
│   │  读数据·分析  │  任务查询    │  做决策·分派任务  │    │
│   │  呈报·提醒   │              │  审查·协调团队    │    │
│   │  ❌ 不做决策  │              │  ✅ 唯一决策者     │    │
│   └──────────────┘              └──────────────────┘    │
│          │                              │               │
│          ▼                              ▼               │
│   ┌──────────────────────────────────────────────┐      │
│   │          全系统数据（产品/库存/销售/财务...）      │      │
│   │          子 Agent 团队（财务/营销/运营...）        │      │
│   └──────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────┘
```

| 维度 | 商务秘书 (Secretary) | CEO Agent (主控官) |
|------|---------------------|-------------------|
| **核心职责** | 数据读取、统计分析、情报呈报、异常提醒 | 战略决策、任务分派、团队协调、结果审查 |
| **数据权限** | **全系统只读**——可读取所有业务数据 | 读写——可修改 PCP、分派任务 |
| **决策能力** | ❌ 不做决策，只提供信息和建议选项 | ✅ 唯一决策者，生成确认卡片 |
| **交互频率** | 高频——随时响应老板查数据、问报表 | 中低频——重大决策、任务分派时介入 |
| **学习目标** | 学习**老板关注什么数据**（老板画像） | 学习**老板如何做决策**（决策偏好） |
| **上下文存储** | 老板关注配置（BAP） | 项目上下文协议（PCP） |
| **常驻位置** | 右下角浮动窗口 | 左侧聊天列表 / ChatPage |

> **一句话总结**：秘书是老板的「眼睛和耳朵」，负责看清现状；CEO Agent 是老板的「大脑和手」，负责判断并执行。

#### 1.3 目标
- **身份重塑**：将右下角浮动 AI 助手重新定位为 **「商务秘书」**（Business Secretary），作为 ProClaw 的**内置 Agent**，出厂即带，无需从市场安装。默认形象为**女性**。
- **职责明确**：全局只读数据分析师，只呈报、不决策。
- **独立上下文**：建立秘书专属的「老板关注配置」（Boss Attention Profile, BAP），让她越用越懂老板需要什么数据。
- **人格化定制**：支持用户右键设置秘书的**名称**和**头像**，打造专属感。
- **语音预留**：在 UI 和架构层面为未来的语音对话功能预留空间。

### 2. 用户故事

#### 2.1 人格化交互
- **作为** 商户老板，我打开 ProClaw 桌面端，右下角出现一个可爱的女孩秘书头像。我右键点击，给她取名「小鹿」，换上一个干练的商务女性头像。从此每天和她对话，就像和自己的专属秘书交流。
- **作为** 新用户，我第一次打开 ProClaw，右下角出现一位干练的女孩秘书，默认名为「小 Pro」。她主动打招呼：「老板好，我是您的商务秘书，可以叫我小 Pro。需要帮您做什么？」

#### 2.2 数据查询与分析
- **作为** 老板，我问秘书：「最近一周销量最好的前 5 个产品是什么？」秘书瞬间拉出排行榜，并附注「其中产品A已经连续 3 周位居榜首，建议关注补货节奏」。
- **作为** 老板，我对秘书说：「对比一下本月和上月的毛利率。」秘书生成对比报表，主动标注出毛利率下降最多的 3 个品类，并提示「如需深入分析，我可以让财务 Agent 出一份专项报告」。
- **作为** 老板，我每天早晨打开 ProClaw，秘书自动推送一条**今日经营简报**：「老板早上好！截至昨天：今日待处理订单 12 笔，库存预警 3 项，昨日营收 ¥8,520，比前天 +5.2%。祝您今天顺利！」

#### 2.3 越用越懂老板
- **作为** 老板，我经常问库存和毛利率，很少问营销数据。几周后秘书的简报自动把「库存预警」和「毛利概览」放在最前面，营销数据则精简为一句话。
- **作为** 老板，我曾让秘书「每周五下午提醒我审审批预算」。从此秘书每周五准点提醒，并附上本周预算执行摘要。
- **作为** 老板，我连续多次在问完销售数据后追问「退货率」。秘书学会了：以后报销售数据时，自动附带退货率。

#### 2.4 不越界、不决策
- **作为** 老板，我问秘书：「库存不足了，帮我自动下单补货吧。」秘书礼貌回应：「老板，下单属于经营决策，我没有这个权限。不过我可以帮您整理好补货清单，您确认后我转交给采购团队处理。您也可以让 CEO Agent 来决策并分派这个任务。」
- **作为** 老板，我问秘书：「你觉得我应该开除哪个员工？」秘书回应：「老板，人事决策我无法参与。不过我可以帮您拉取团队绩效数据供您参考。」

### 3. 功能设计

#### 3.1 身份定位

| 属性 | 说明 |
|------|------|
| **Agent 类型** | 内置 Agent（Built-in），非市场安装 |
| **Agent ID** | `builtin:secretary` |
| **显示名称** | 默认为「小 Pro」，用户可自定义 |
| **默认头像** | ProClaw 品牌默认秘书形象（**女性**，商务风格） |
| **性别** | 女（默认，不可切换） |
| **数据权限** | **全局只读**——可查询所有模块数据，不可修改、不可创建 |
| **决策权限** | ❌ 无——不替代 CEO Agent 的决策职能 |
| **系统角色 prompt** | 「你是 ProClaw 的商务秘书，一位专业、温暖、高效的数据分析师。你可以读取老板的全部经营数据，提供统计分析和情报呈报。你**不做决策**——决策是 CEO Agent 的职责；你**不提敏感建议**（如人事、定价策略）；你的价值在于让老板看清数据全貌，辅助他做出判断。你的语气亲切但不失分寸，偶尔展现一点小幽默。」 |
| **常驻位置** | 桌面端所有页面右下角（FloatingAgentChat 组件位置） |

#### 3.2 核心能力矩阵

| 能力 | 说明 | 示例 |
|------|------|------|
| **数据查询** | 支持自然语言查询全系统数据 | 「上月销售额」「库存<10的产品」「供应商A的采购记录」 |
| **统计分析** | 排行、对比、趋势、占比 | 「近3月销售趋势图」「品类毛利占比」「环比增长率」 |
| **异常预警** | 主动监测并提醒异常指标 | 「产品A库存低于安全线」「今日退货率异常升高」 |
| **定时简报** | 按老板偏好推送每日/每周经营摘要 | 每日早安简报、周五周报、月初月度对比 |
| **关联洞察** | 发现数据间的隐含关联 | 「销量上升的同时退货率也在上升，可能与某批次质量有关」 |
| **信息转交** | 将老板意图打包转给 CEO Agent | 「把补货清单和预算上限转给 CEO，让它分派采购任务」 |

> **能力边界**：秘书可做的（分析、呈报、提醒）vs 不可做的（定价、采购、人事、战略决策）。如果老板提出的需求越界，秘书应委婉拒绝并引导至 CEO Agent 或对应子 Agent。

#### 3.3 老板关注配置（Boss Attention Profile, BAP）

秘书的核心差异化能力——**记住老板关心什么，并越来越精准地服务**。

##### 3.3.1 BAP 是什么

BAP 是秘书专属的上下文存储，记录老板的数据偏好，存储在 SQLite 中（区别于 CEO Agent 的 PCP）：

```sql
CREATE TABLE secretary_bap (
  id TEXT PRIMARY KEY,
  profile_type TEXT NOT NULL,  -- 'kpi_preference', 'time_routine', 'alert_threshold', 'focus_area', 'custom_rule'
  key TEXT NOT NULL,
  value JSON NOT NULL,
  confidence REAL DEFAULT 0.5, -- 置信度（0-1），越高越确定是老板的稳定偏好
  source TEXT DEFAULT 'observed', -- 'observed'(观察习得) / 'explicit'(老板明确设置) / 'inferred'(推理)
  last_matched_at INTEGER,
  created_at INTEGER DEFAULT (unixepoch()),
  updated_at INTEGER DEFAULT (unixepoch())
);
```

##### 3.3.2 BAP 包含什么

| 配置类型 | 说明 | 示例 |
|----------|------|------|
| **关注指标 (KPI)** | 老板高频查看的指标及优先级 | `[{kpi: '毛利率', priority: 1}, {kpi: '库存周转', priority: 2}]` |
| **时间惯例 (Routine)** | 老板习惯的查看节奏 | `{daily_brief: '09:00', weekly_report: '周五 17:00', monthly_compare: '月初'}` |
| **预警阈值 (Alert)** | 老板设定的自动预警条件 | `{stock_low: 10, margin_drop: 5%, refund_rate_high: 8%}` |
| **关注领域 (Focus)** | 老板重点关注的产品/品类/客户/供应商 | `{products: ['A001','A003'], categories: ['手机配件'], suppliers: ['深圳XX']}` |
| **报告偏好 (Report)** | 老板偏好的报表格式和内容结构 | `{chart_first: true, include_comparison: true, detail_level: 'summary'}` |

##### 3.3.3 BAP 如何学习

秘书通过三种方式逐步构建老板画像：

**① 被动观察（Observed）**——核心学习方式
- 记录老板每次查询的关键词、模块、时间。
- 例如：老板最近 20 次查询中，14 次涉及「库存」和「毛利」→ BAP 自动将这两个指标置信度提升至 0.9。
- 例如：老板每周一早上习惯问「上周汇总」→ 秘书学会每周一自动推送周报。

**② 显式设置（Explicit）**——老板可手动配置
- 右键菜单新增「📊 关注设置」，打开 BAP 配置面板。
- 老板可手动设置：我想关注哪些指标、预警阈值多少、简报推送时间。

**③ 推理学习（Inferred）**——关联行为推断
- 例如：每次旺季（春节/双11）前，老板都频繁查库存 → 秘书在下一个旺季前提早 3 天预警。
- 例如：老板问完「销售额」后，80% 的概率会追问「退货率」→ 秘书默认将两者打包呈报。

##### 3.3.4 BAP 带来的体验变化

| 时间 | 阶段 | 秘书表现 |
|------|------|----------|
| 第 1 天 | 空白 | 通用欢迎语，万能但不精准 |
| 第 1 周 | 初步观察 | 开始记住老板常查的模块，快捷指令动态调整 |
| 第 1 月 | 画像成形 | 每日简报精确命中老板关注点，预警阈值自动校准 |
| 长期 | 深度默契 | 老板问一句她能补三句，关联推理主动呈报，成为真正的「懂老板的秘书」 |

##### 3.3.5 BAP 与 CEO Agent 的 PCP 的关系

- **BAP（秘书维护）**：记录「老板**关注什么数据**」——微观、高频、数据偏好。
- **PCP（CEO Agent 维护）**：记录「公司的**战略方向和约束**」——宏观、低频、决策框架。
- **交互**：秘书可将 BAP 中发现的重大异常推送给 CEO Agent（如「毛利率连续 3 周下滑，建议纳入 PCP 关注」），但秘书不直接修改 PCP。

#### 3.4 碰壁处理——当老板要求越界时

秘书必须有一套得体的「碰壁话术」，拒绝时给出替代方案：

| 老板越界请求 | 秘书回应 |
|-------------|----------|
| 「帮我下采购单」 | 「老板，下单需要决策权限，我帮您整理好补货清单，您确认后我转给 CEO Agent 分派采购任务。」 |
| 「这个产品定什么价」 | 「定价涉及市场策略，建议您咨询 CEO Agent。我可以提供同类产品市场价参考和成本分析，帮您做判断。」 |
| 「帮我开除XX」 | 「人事决策不在我的能力范围内。需要我帮您调取 XX 的绩效数据供参考吗？」 |
| 「你觉得这个营销方案行不行」 | 「我不做主观判断哦～不过我可以帮您拉取同类活动的历史数据，看看效果如何。」 |

> **设计原则**：拒绝 + 替代方案 + 引导至正确入口。不让老板感到被堵住，而是被引导。

### 4. 个性化定制

#### 4.1 右键上下文菜单

在秘书**头像区域**（包括浮动按钮的 FAB 和展开面板的头像）上右键时，弹出上下文菜单：

```
┌─────────────────────────┐
│  🖼 更换头像             │
│  ✏️ 修改名称             │
│  ─────────────────────  │
│  📊 关注设置             │  ← 新增：打开 BAP 配置面板
│  ─────────────────────  │
│  🎤 语音通话 （即将上线） │  ← 灰态，tooltip 显示"语音通话功能即将上线"
│  ─────────────────────  │
│  ℹ️ 关于秘书             │
└─────────────────────────┘
```

**交互细节**：
- 右键菜单位置跟随鼠标，使用 MUI `Menu` 组件 + `contextmenu` 事件。
- 「更换头像」→ 弹出头像选择器（见 4.2）。
- 「修改名称」→ 弹出名称编辑对话框（见 4.3）。
- 「📊 关注设置」→ 打开 BAP 配置面板（见 4.4）。
- 「语音通话」永久灰态，hover 显示 tooltip：「语音通话功能即将上线，敬请期待」。
- 「关于秘书」→ 弹出简短说明对话框。

#### 4.2 头像系统

##### 4.2.1 预设头像库

内置一组预设秘书头像（6-8 个），风格统一为商务插画风，全部为女性形象：

| 编号 | 描述 | 风格 |
|------|------|------|
| default | ProClaw 品牌默认秘书形象（女孩） | 女性、短发、商务西装 |
| avatar_01 | 知性风格 | 女性、眼镜、微笑 |
| avatar_02 | 干练风格 | 女性、马尾、干练 |
| avatar_03 | 亲和风格 | 女性、圆脸、温暖 |
| avatar_04 | 专业风格 | 女性、职场、自信 |
| avatar_05 | 活力风格 | 女性、年轻、朝气 |
| avatar_06 | ProClaw 猫头鹰吉祥物 | 品牌IP形象 |

##### 4.2.2 头像选择器 UI

- 弹出 Dialog，展示所有预设头像的网格（3 列布局）。
- 当前选中的头像有蓝色边框 + ✓ 标记。
- 底部有说明文字：「未来版本将支持上传自定义头像」。
- 选择后立即生效，无需额外确认。

##### 4.2.3 头像存储

- 头像选择结果存储在 `localStorage`：`key: proclaw-secretary-avatar, value: 'default' | 'avatar_01' | ...`
- 默认值为 `'default'`。
- 未来可扩展为存储头像文件的 base64 或文件路径。

#### 4.3 名称自定义

##### 4.3.1 修改流程

- 右键菜单 → 「修改名称」→ 弹出小型 Dialog。
- 包含一个 TextField，预填当前名称。
- 限制：2-10 个字符，支持中文/英文/数字/emoji。
- 点击「确认」后立即生效，聊天面板标题和欢迎语同步更新。
- 若修改成功，秘书回复一条消息：「好的老板，以后就叫我『{新名字}』吧！」

##### 4.3.2 名称存储

- 存储在 `localStorage`：`key: proclaw-secretary-name, value: '小 Pro'`
- 默认值为 `'小 Pro'`。

#### 4.4 BAP 配置面板

- 右键菜单 → 「📊 关注设置」→ 弹出侧边面板或 Dialog。
- 面板分为四个区域：
  1. **我关注的指标**：勾选/拖拽排序 KPI（库存、营收、毛利、退货率……），秘书按优先级呈报。
  2. **自动预警**：设置各指标的预警阈值，支持开关。
  3. **简报推送**：设置每日简报时间、是否推送周报/月报。
  4. **学习记录**：展示秘书从老板行为中学到的偏好（只读），老板可「重置学习」清空观察数据。

#### 4.5 语音通话预留

| 预留内容 | 实现方式 |
|----------|----------|
| **UI 入口** | 右键菜单「🎤 语音通话」项，永久灰态 + tooltip |
| **未来通话按钮** | 浮动面板标题栏预留麦克风图标按钮位置（当前隐藏） |
| **数据接口** | `SecretaryVoiceConfig` 类型预留 |
| **音频输出预留** | 消息气泡旁预留「🔊 朗读」按钮位置（当前隐藏） |
| **语音唤醒预留** | 未来可在设置中增加「语音唤醒词」配置 |

### 10. 风险与备选

- **BAP 冷启动**：新用户无历史数据时，BAP 为空，秘书表现泛化。应对：提供预设模板（如「零售行业关注模板」「餐饮行业关注模板」），用户可在关注设置中一键套用，加速冷启动。
- **过度学习偏差**：如果老板某段时间集中关注某个问题（如临时盘点库存），BAP 可能过度拟合。应对：BAP 采用时间衰减权重，近 7 天数据权重更高，30 天前数据权重减半。同时老板可在面板中手动锁定/解锁偏好。
- **全系统只读的性能**：秘书的全局只读权限意味着每次查询可能跨多个数据表。应对：Rust 后端实现查询缓存 + 增量刷新；频繁查询的指标（库存总量、今日营收）每 5 分钟缓存一次。
- **与 CEO Agent 职责混淆**：用户可能不区分秘书和 CEO，向秘书下达决策指令。应对：碰壁话术清晰引导，配合 UI 层面视觉区分（秘书面板右下角、CEO 在左侧聊天列表），降低混淆。
- **头像设计资源延迟**：若设计师未及时产出头像 SVG，初期使用 emoji 或不同颜色的 MUI Avatar 占位，头像选择器 UI 框架先行搭建。
- **语音功能依赖**：TTS/STT 能力依赖后续大模型接入和浏览器 Web Speech API，当前仅预留 UI 和类型结构。
