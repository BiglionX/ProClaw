## 一、背景与现状分析

### 1.1 页面结构现状（2026-06-05）

当前 AI 团队页面（`TeamsPage.tsx`，1488 行）包含两个分页：

| 分页 | 内容 | 状态 |
|------|------|------|
| **Agent** | 3 张原生 Agent 卡片（CEO Agent、商务秘书、AI 客服） | ✅ 初步可用 |
| **AI Team** | 团队卡片列表 + 过滤 Tab + AI 推荐 + CRUD 对话框 | ⚠️ 存在 UI 缺陷 |

### 1.2 AI Team 分页现存 UI 缺陷

| # | 问题 | 严重度 | 具体表现 |
|---|------|--------|----------|
| 1 | **页面级 Tab 设计粗糙** | 🔴 高 | Agent\|AI Team 用自定义 Box 模拟 Tab，与 MUI Tabs 风格不统一；Tab 放置在标题同一行，挤占空间 |
| 2 | **标题栏信息过载** | 🟡 中 | 单行 Header 塞了：标题 + 2 个 Tab + 5 个操作按钮（AI 推荐+刷新+导入+创建+隐藏文件输入），视觉混乱 |
| 3 | **操作按钮缺乏分组** | 🟡 中 | 主要操作（创建团队）与次要操作（刷新、导入）混排，没有层级区分 |
| 4 | **团队卡片信息密度过高** | 🔴 高 | 单张卡片塞了 8 种信息：渐变头像、名称、状态 Chip、版本 Chip、成员数 Chip、AvatarGroup、描述、标签、分类+来源、6 个操作图标按钮 |
| 5 | **卡片操作过多** | 🟡 中 | 6 个 IconButton（对话/详情/编辑/导出/发布/删除）在卡片底部一字排开，难点击、易误触 |
| 6 | **推荐区域侵入式嵌入** | 🔴 高 | AI 推荐卡片直接插入在内容流顶部，有推荐时占用额外空间；卡片无操作引导 |
| 7 | **Tab 筛选与实体列表脱节** | 🟡 中 | 顶部 Tabs（全部/已发布/私有）与团队卡片列表之间缺少搜索、排序功能，用户无法快速定位 |
| 8 | **缺少团队统计概览** | 🟡 中 | 页面顶部没有"共 X 个团队 / X 个已发布 / X 个 Agent 成员"等摘要信息 |
| 9 | **空状态设计简约但可用** | 🟢 低 | 空状态 Paper 有图标+文案+按钮，功能完整但缺少视觉吸引力 |
| 10 | **秘书 Agent 导航目标不合理** | 🟡 中 | 商务秘书卡片点击后跳转到 `/teams`（即相同页面），产生死循环 |
| 11 | **移动端响应式表现** | 🟡 中 | 3 列 Grid 在小屏降为 1 列时卡片宽度正常，但操作按钮过小 |
| 12 | **Card 呼吸动画无开关** | 🟢 低 | 已发布团队卡片有呼吸边框动画，但缺少动画控制，大量卡片时可能影响性能 |

## 二、设计目标

1. **语义化页面结构**：标题 → 统计概览 → 搜索/筛选 → 团队列表，建立清晰的视觉动线
2. **减少认知负荷**：每张卡片展示 4-5 个核心信息维度，次要操作折叠到详情
3. **操作路径优化**：高频操作（对话、创建团队）置于表层，低频操作（导出、删除）收入菜单
4. **Agent 分页可扩展**：Agent 卡片模式支持后续新增原生 Agent
5. **AI 推荐自然融入**：推荐作为提示条而非侵入式卡片
6. **与全局 UI 升级对齐**：使用 v11.0 PRD 定义的设计 Token 和组件风格

## 三、功能改造方案

### 3.1 Header 区域重构

#### 3.1.1 当前布局（问题）

```
┌──────────────────────────────────────────────────────────────┐
│  AI 团队              [Agent│AI Team]  [AI推荐][刷新][导入][创建]  │
└──────────────────────────────────────────────────────────────┘
```

#### 3.1.2 改造后布局

```
┌──────────────────────────────────────────────────────────────┐
│  AI 团队                                    [创建团队]        │  ← 第一行：标题 + 主按钮
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Agent  │  AI Team                                   │   │  ← 第二行：MUI Tabs 组件
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

**改动要点**：
- 将页面级 Tab（Agent\|AI Team）从标题行独立到第二行，使用标准 MUI `Tabs` 组件
- 标题行只保留标题 + 主要操作按钮（创建团队）
- 次要操作（刷新、导入、AI 推荐）移到 AI Team Tab 内容区右上角或工具栏

### 3.2 AI Team Tab 内容区重构

#### 3.2.1 统计概览栏（新增）

在 Tab 内容顶部增加轻量统计条：

```
┌──────────────────────────────────────────────────────────────┐
│  📊 共 3 个团队  ·  1 个已发布  ·  19 个 Agent 成员           │
└──────────────────────────────────────────────────────────────┘
```

- 使用 `Typography` + `Chip` 组合，视觉轻量
- Agent 成员数为所有团队 `members.length` 之和

#### 3.2.2 搜索与筛选栏（新增）

```
┌──────────────────────────────────────────────────────────────┐
│  [🔍 搜索团队名称...]  [全部 ▼]  [排序 ▼]  [刷新][导入][AI推荐]   │
└──────────────────────────────────────────────────────────────┘
```

**改动要点**：
- 新增 `TextField` 搜索框，按团队名称/描述/标签过滤
- Tab 筛选（全部/已发布/私有）保留，改为 MUI `ToggleButtonGroup`
- 新增排序选项（创建时间/更新时间/成员数量）
- 次要操作（刷新、导入、AI 推荐）从 Header 移到此处

#### 3.2.3 AI 推荐区域改造

**当前**：推荐卡片嵌入内容流，与团队卡片混排

**改造后**：推荐信息以 Alert/Banner 形式展示在内容区顶部

```
┌──────────────────────────────────────────────────────────────┐
│  💡 AI 根据你的业务数据，推荐创建「库存专业助手」团队  [查看详情 →]  [×]  │
│  配置：库存优化师 + 销售预测分析师 + 业务分析师  共 5 个 Agent      │
└──────────────────────────────────────────────────────────────┘
```

- 使用 MUI `Alert` 风格的 Banner，可关闭（localStorage 持久化）
- 点击「查看详情」打开推荐对话框（保持现有 `recommendOpen` 逻辑）
- Banner 在已存在推荐团队时自动隐藏

#### 3.2.4 团队卡片 UI 精简

**当前卡片**（信息过载）：
```
┌──────────────────────────────┐
│ 🟪  名称               v1.0 │
│     [已发布] [5成员] 👤👤👤  │
│     描述文字两行...          │
│     [标签1][标签2][标签3]    │
│     分类 · 来源             │
│     💬👁✏️📤📌🗑️            │  ← 6 个按钮
└──────────────────────────────┘
```

**改造后卡片**（精简）：
```
┌──────────────────────────────┐
│ 🟪  名称          [已发布] │
│     描述文字一行...         │
│     [标签1] [标签2]        │
│     👤👤👤 +2  5成员       │
│                        💬 ⋮ │  ← 对话 + 更多菜单
└──────────────────────────────┘
```

**改动要点**：

| 元素 | 当前 | 改造后 | 说明 |
|------|------|--------|------|
| 渐变头像 | 46x46 | 40x40 | 缩小，按分类分配不同渐变色 |
| 名称 | 显示 | 显示 | 不变 |
| 版本 Chip | 显示 | 取消 | 移入详情对话框 |
| 状态 Chip | 居中 | 右上角 | 移到卡片右上角 |
| 成员数 Chip | 显示 | AvatarGroup 旁 | 移到头像阵列右侧 |
| 描述 | 2 行 | 1 行 | 缩减留白 |
| 标签 | 最多 4 个 | 最多 2 个 | 其余折叠到详情 |
| 分类+来源 | 底部 | 取消 | 移入详情对话框 |
| 操作按钮 | 6 个 | 2 个 | 💬 对话 + ⋮ 更多（含编辑/导出/发布/删除） |

**卡片交互增强**：
- 悬停时边框高亮 + 轻微上移（已有，保持）
- 点击卡片主体 → 打开团队详情对话框（当前需点击👁图标）
- 呼吸动画保留（已发布团队），新增 `prefers-reduced-motion` 支持

#### 3.2.5 团队详情对话框增强

当前详情对话框（`TeamDetailDialog`）功能完整，但视觉可优化：
- 信息区域用深色 Paper 填充，在浅色主题下对比度低
- 工作日志 Tab 无刷新按钮
- 成员列表只显示首字母，无对应 Agent 头像

**改造要点**：
- 使用 MUI `Card` 替代 Paper 作为信息容器，提升视觉层次
- 工作日志 Tab 增加手动刷新按钮
- 成员列表 Avatar 使用 `getDefaultAgentAvatar`（复用 `contactService` 的逻辑）显示 emoji 头像
- 成员列表增加「发起对话」快捷按钮（复用当前 `onChat` 回调）

### 3.3 Agent 分页增强

#### 3.3.1 当前问题

| 问题 | 描述 |
|------|------|
| 商务秘书导航死循环 | 点击秘书卡片 → `/teams` → 回到同页面 |
| 无状态反馈 | 卡片点击后无任何加载/选中状态 |
| 缺少配置入口 | CEO Agent 卡片直接跳转到 `/settings`，路径不直观 |
| 卡片数量固定 | 硬编码 3 个 Agent，后续新增需改代码 |

#### 3.3.2 改造方案

- **商务秘书**：点击后打开内置的秘书对话面板（`BapSettingsPanel`）而非跳转页面
- **CEO Agent**：点击后打开配置对话框（集成 `PreferenceSettings` / `CompanyConfigManager` 组件）而非跳转 `/settings`
- **AI 客服**：点击后导航到 `/customer-service`（保持当前行为）
- **状态反馈**：点击卡片后显示加载指示器，避免重复点击
- **可扩展设计**：`AGENTS` 数据数组支持从配置文件/后端动态加载

#### 3.3.3 Agent 卡片样式微调

- 卡片标题下方增加「状态指示器」（在线/配置中/离线）
- CEO Agent 加「主控官」徽章
- 商务秘书加「可对话」徽章
- 悬停时显示功能入口按钮（替代全卡片点击跳转）

### 3.4 对话框整合

#### 3.4.1 创建团队对话框（`CreateTeamDialog`）

**现状**：基本可用，仅支持输入名称、描述、分类、标签

**改造**：
- 增加成员配置区域（从编辑对话框复用成员管理逻辑）
- 增加预设模板选择（AI 经营团队、电商客服团队等）
- 创建成功后自动同步群组（当前已有 `syncAITeamGroups` 调用）

#### 3.4.2 导入对话框（`importDialogOpen`）

**现状**：独立 Dialog，含市场导入和本地文件导入两个按钮

**改造**：
- 合并到创建团队流程中作为导入选项
- 或改为「创建或导入团队」统一的浮动入口按钮

## 四、交互流程

### 4.1 用户进入 AI Team 分页

```
[用户点击侧边栏"AI 团队"] 
    → TeamsPage 加载
    → 默认显示 Agent 分页（activePageTab=0）
    → 加载团队列表（loadTeams）
    → 同步群组（syncAITeamGroups）
    → 如果有推荐且未关闭 → 显示推荐 Banner
    → 渲染团队卡片列表或空状态
```

### 4.2 用户创建团队

```
[点击"创建团队"按钮]
    → 弹出 CreateTeamDialog（增强版）
    → 填写名称/描述/选择模板
    → 提交 → safeInvoke('create_team')
    → 成功 → 刷新列表 → 同步群组
    → Snackbar 提示
```

### 4.3 用户与团队对话

```
[点击团队卡片「💬」按钮]
    → navigate('/chat/' + buildGroupId(team.id))
    → ChatPage 加载该团队群聊
    → 显示种子消息 + 团队成员列表
```

### 4.4 AI 推荐流程（不变，仅 UI 改进）

```
[点击"AI 推荐"按钮或加载时检测到推荐]
    → 显示 Banner（若已有推荐）
    → 或打开推荐对话框
    → 采集业务数据 → 规则引擎/LLM 生成
    → 展示推荐结果
    → 用户选择"在 ProClaw 中创建"或"在 NvwaX 中创建"
```
