项目名称：ProClaw-Light
定位：基于 ProClaw 进销存版裁剪，专门面向县区小微商家的极简智能经营助手，聚焦获客、履约、客户管理，内置 AI Team 辅助运营。

1. 与标准版 ProClaw 的差异
功能模块	标准 ProClaw（进销存版）	ProClaw-Light
进销存	完整（采购、销售、库存、报表）	轻量版：保留商品管理 + 销售单 + 库存流水。隐藏采购模块（可使用团购同步补货）。
财务管理	完整记账、发票	仅收入/支出记录，无复杂报表。
多角色权限	支持（多角色细粒度）	仅老板 + 员工角色（员工可看订单和商品，不可修改价格）。
AI Team 集成	通过市场安装（可选）	✅ 预装三大 Team（新媒体、团购、小程序），开机引导一键激活。
设备直连	支持	移除（所有用户均使用同一台电脑，无需多设备）。
移动端 App	独立 App	不需要，所有操作在桌面端完成。
云备份	可选 Pro	默认本地备份（U盘或自动备份到第二硬盘）。
计费	按 token 消耗	免费使用基础功能，高级 AI（如视频生成）按次扣 token。
媒体库	无独立模块（仅商品图片上传）	✅ 独立媒体库管理，支持图片/视频分类、标签、跨平台复用。
问答库	无	✅ 客服场景问答库，商户维护常见客户问题与标准回答。
资料库	无	✅ 商户资料库（说明文档、报价单、政策模板等）。
2. 界面设计（基于 ProClaw 进销存版 UI 裁剪）
保留进销存版现有界面框架（Sidebar + TopBar + 内容区），不做全新 UI 重设计。

变更点：
- 左侧导航调整为：商品、销售、库存、客户、统计、AI 团队、媒体库、问答库、资料库
- 隐藏/移除：采购管理、财务管理（完整版）、设备配对
- 权限裁剪：仅保留老板 + 员工两种角色
- 仪表板首页：保留现有数据看板，额外展示三大预装 AI Team（新媒体运营、本地团购、小程序商城）的运行状态卡片

一键操作：
- "一键同步商品到小程序"
- "一键发布今日短视频"
- "一键打印团购订单"

界面复用策略：
- 商品管理 → 复用现有 ProductsPage，默认「简单商品库」模式
- 销售管理 → 复用现有 SalesPage，新增平台来源筛选（本地/团购/小程序）
- 库存管理 → 复用现有 InventoryPage，保留库存流水
- 客户管理 → 复用现有客户模块，增强跨平台订单合并
- 统计看板 → 复用现有 Dashboard，增加平台占比和 AI Team 状态卡片
- AI 团队 → 复用现有 TeamsPage，预装显示三大 Team
- 设置中心 → 复用现有 SettingsPage，裁剪不需要的配置项

3. 关键功能详细说明
3.1 商品管理（复用现有 ProductsPage，默认轻量模式）
商品字段：名称、价格、库存、图片、分类（可自定义）

复用进销存版 ProductsPage，默认启用「简单商品库」模式（关闭 SPU/SKU 电商模式）。

支持从 Excel 批量导入

支持扫码枪添加商品（条码扫描）

3.2 销售管理（复用现有 SalesPage，聚合视图）
统一下拉展示：本地销售单、团购订单、小程序订单

复用进销存版 SalesPage，新增「平台来源」筛选字段。

支持按平台筛选、状态筛选（待发货、已发货、已完成）

点击订单可打印小票（连接热敏打印机）

3.3 库存管理（复用现有 InventoryPage，保留流水）
保留进销存版库存流水查询功能

库存预警（低于阈值标红）

隐藏采购入库模块（替代方案：团购订单到货自动入库）

3.4 客户管理（复用现有客户模块，增强）
自动合并同一客户在不同平台的订单记录

显示客户总消费、最近购买时间

支持发送短信（需购买短信包）

3.5 数据看板（复用现有 Dashboard）
今日销售额、订单数、各平台占比

热销商品排行榜

库存预警（低于阈值标红）

三大 AI Team 运行状态卡片（新媒体运营、本地团购、小程序商城）

3.6 媒体库管理（新增模块）
定位：统一的图片/视频资产管理中心，供各平台和 AI Team 复用素材。

核心功能：
- 图片/视频上传（支持拖拽、批量上传）
- 分类管理（商品图、海报素材、短视频、Banner 等自定义分类）
- 标签系统（按用途、平台、商品关联打标签）
- 缩略图网格预览，按分类/标签/日期筛选搜索
- 跨模块引用：商品编辑可直接从媒体库选图，社媒 Agent 发帖时可引用媒体库素材

存储：本地文件系统（Tauri fs），媒体元数据存 SQLite。

3.7 问答库（新增模块，客服场景）
定位：商户维护的「客户常见问题 + 标准回答」知识库，供客服场景和 AI 助手调用。

与现有产品 FAQ 的区别：
- 现有 FAQ：面向 ProClaw 用户，解答软件使用问题
- Light 问答库：面向商户的客户，解答商品/服务相关问题

核心功能：
- 问答对管理（问题、答案、分类、标签）
- 快速检索与模板插入
- 支持分类：商品咨询、配送退换、售后服务、促销活动等
- 与 AI 助手联动：客户提问时 AI 自动匹配问答库给出标准回答
- 支持导出为团购平台/小程序的快捷回复模板

复用策略：基于现有 faqService.ts 和 FAQManagementPage.tsx 扩展，新增「业务场景」维度。

3.8 资料库（新增模块）
定位：商户的内部知识库，存储业务相关文档和参考资料，AI 助手可检索回答客户提问。

核心功能：
- 文档上传与管理（支持 PDF、Word、Excel、图片、文本等格式）
- 分类管理（产品说明书、报价单、配送政策、营销模板、合同范本、证照资质等）
- 全文搜索（基于文件内容的关键词检索）
- 与 AI 助手联动："土鸡蛋的退换货政策是什么"→ AI 检索资料库中的配送政策 → 自动生成回答
- 一键引用：在处理订单、回复客户时快速引用资料库内容

存储：本地文件系统存储原文件 + SQLite 存储元数据和文本索引。

3.9 AI 助手（悬浮球，四库联动）
任何时候点击悬浮球，可用自然语言提问，AI 基于四库知识底座回答：

- 数据类：「上个月抖音来的订单有多少？」（查订单数据）
- 创作类：「帮我生成一条小红书文案，商品是土鸡蛋。」（商品库 + 媒体库素材）
- 客服类：「客户问土鸡蛋能不能退货，帮我回复。」（自动检索问答库 + 资料库）
- 知识类：「查一下我们店的退换货政策。」（检索资料库）

AI 知识底座 = 商品库 + 订单数据 + 问答库 + 资料库 + 媒体库，四库联动。

使用 DeepSeek 模型，按 token 扣费（预充值）

4. 安装与首次使用流程
老板下载安装 ProClaw-Light.exe（基于进销存版构建包）

启动后，安装向导引导：

1. 选择店铺类型（餐饮、零售、服务、生鲜等）
2. 是否已有商品数据（导入 Excel 或空白开始）
3. 是否已有抖音 / 美团 / 小程序账号（如无，引导注册）
4. 自动从 Nuwax 安装「MicroBiz AI Team Suite」（预装新媒体运营、本地团购、小程序商城三大 Team）
5. 引导初始化内容库：
   - 上传常用商品图片到媒体库
   - 设置几条高频客服问答
   - 上传配送/退换货政策到资料库
6. 进入主界面，弹出新手任务卡片：
   - 同步第一个商品
   - 上传第一组素材到媒体库
   - 发布第一条抖音 / 团购活动
   - 处理第一个订单

5. 硬件适配
支持连接 USB 小票打印机（通过 Tauri 调用系统打印或使用 node-escpos）

支持 条码扫描枪（监听键盘输入）

支持 语音播报（新订单朗读，Tauri 调用 speech_synthesis）

6. 技术栈
沿用 ProClaw 基础：Tauri 2 (Rust) + React + SQLite

增加 Node.js 后端（可选）用于对接外部 API（可通过 Tauri 的 sidecar 模式）

打印模块：使用 escpos 库

语音：Tauri 调用 speech_synthesis

7. 分发与更新
提供官网下载（exe/msi 安装包）

支持自动静默更新（tauri updater）

数据备份提醒：每周提示老板插入 U 盘备份

8. 收费模式
基础功能永久免费（商品、订单、本地报表、手动发布）

AI 按量付费：充值 token，用于文案生成、视频合成、数据分析高级功能

增值服务：代运营（付费人工辅助）—— 可选

9. 风险与应对
平台 API 变动：每个 Agent 独立维护，由 Nuwax 统一更新

电脑损坏：定期备份提醒 + 可选 Pro 云备份

老板不会打字：支持语音输入（浏览器内置语音识别）
