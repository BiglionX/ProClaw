## 一、背景与现状分析

### 1.1 当前问题

| 问题 | 严重度 | 描述 |
|------|--------|------|
| **AI 任务不可见** | 🔴 高 | 当前 CEO Agent 分派的任务仅能通过 AI Team 群聊中的事件消息感知，Boss 无法主动查看全局任务状态 |
| **无集中监控入口** | 🔴 高 | 任务分散在群聊消息、Agent 对话框等多处，缺少统一的"任务看板"来掌握各 Agent 的工作进展 |
| **统计缺失** | 🟡 中 | Boss 无法快速了解：今天派了多少任务、完成率如何、哪些 Agent 最忙/最闲、是否有关键任务积压 |
| **与数据中心割裂** | 🟡 中 | 数据中心已有业务分析、利润表、现金流量表三个经营维度，但缺少"AI 人力/算力"维度的概览 |

### 1.2 现有能力基础

| 能力 | 位置 | 状态 |
|------|------|------|
| CEO 任务 CRUD | `src/lib/ceoController.ts` → `proclawTasks` | ✅ 已实现 |
| 任务统计 API | `ceo_get_task_stats` (Tauri command) | ✅ 已实现 |
| Agent 列表 API | `get_installed_agents` (Tauri command) | ✅ 已实现 |
| 任务类型 | `CeoTaskRecord`: id, task_id, assigned_agent_id, type, description, expected_output, priority, status, result, created_at, deadline, completed_at | ✅ 已定义 |
| 任务状态 | pending / in_progress / completed / failed / cancelled | ✅ 已定义 |

### 1.3 定位

> **「AI 任务概览」是数据中心的第四个维度**——在业务分析（经营指标）、利润表（财务健康）、现金流量表（资金流动）之外，补充「AI 劳动力」视角，让 Boss 一站式掌握"人的生意"和"AI 的产出"。

### 2.3 模块详细设计

#### 2.3.1 顶部统计卡片行

| 卡片 | 数据字段 | 颜色 | 附加信息 |
|------|---------|------|----------|
| 总任务数 | `TaskStats.total` | 品牌红 `#FF3B30` | 环比变化百分比（较上月） |
| 进行中 | `TaskStats.in_progress` | AI 紫 `#6366F1` | — |
| 已完成 | `TaskStats.completed` | 数据绿 `#10B981` | 完成率 = completed / total × 100% |
| 失败 | `TaskStats.failed` | 警示金 `#F59E0B` | — |
| 待处理 | `TaskStats.pending` | 中性灰 `#6B7280` | — |

**样式规范**：
- 复用现有 `StatCard` 组件
- 数字使用 `StatMedium` 字体（24px, fontWeight 700），与业务分析卡片保持视觉一致性
- 附加信息使用 Caption 字体（12px）

#### 2.3.2 图表区域（双列布局）

**左：各 Agent 任务分布（横向柱状图）**

```
┌──────────────────────────────┐
│ 📊 各 Agent 任务分布         │
│                              │
│ 新媒体运营  ████████████ 15  │
│ 数据分析    ██████████   12  │
│ 财务管理    ████████      9  │
│ 任务管理    ████          4  │
│ 客户关系    ██            2  │
└──────────────────────────────┘
```

- 数据源：`proclawTasks.list()` 结果按 `assigned_agent_id` 分组计数
- 柱状条使用品牌红到 AI 紫的渐变 `linear-gradient(90deg, #FF3B30, #6366F1)`
- Y 轴为 Agent 名称（截断 + tooltip 显示全名），X 轴为任务数
- 无数据时显示空状态："暂无 Agent 任务数据"

**右：近 30 天任务趋势（折线图）**

```
┌──────────────────────────────┐
│ 📈 近30天任务趋势            │
│                              │
│  完成 ── 绿线 #10B981       │
│  新增 ── 红线 #FF3B30       │
```

- 三条折线：每日新增任务（created_at 计数）、每日完成任务（completed_at 计数）、每日失败任务
- 使用 Recharts `LineChart`，与业务分析的销售趋势图风格一致
- 无数据时显示空状态

#### 2.3.3 任务列表

**筛选栏**：
- 状态筛选：全部 / 待处理 / 进行中 / 已完成 / 失败 / 已取消（下拉选择）
- Agent 筛选：全部 / 按已安装 Agent 列表（下拉选择）
- 搜索框：按任务描述关键词搜索（前端过滤）

**表格列定义**：

| 列名 | 数据来源 | 宽度 | 备注 |
|------|---------|------|------|
| 状态 | `status` | 80px | 带颜色圆点 + 中文标签 |
| 任务描述 | `description` 或 `type` | auto | 主要信息列，过长截断 |
| 指派 Agent | `assigned_agent_id` | 140px | 映射到 Agent 名称 |
| 优先级 | `priority` (1-3) | 80px | ⭐⭐ 形式展示 |
| 创建时间 | `created_at` | 100px | 格式化为相对时间（如"2小时前"）或日期 |

**样式规范**（遵循数据中心 Tab 页样式统一规范）：
- 表格字体统一 `0.875rem`
- 状态列颜色映射：
  - 待处理 `#6B7280`（灰）
  - 进行中 `#6366F1`（AI 紫）
  - 已完成 `#10B981`（数据绿）
  - 失败 `#EF4444`（红）
  - 已取消 `#9CA3AF`（浅灰，删除线）
- 表头 `fontWeight: 600`

**分页**：每页 20 条，使用 MUI TablePagination

**空状态**：
```
┌──────────────────────────────────┐
│          🤖                      │
│    暂无 AI 任务记录              │
│  通过 AI Team 向 CEO Agent       │
│  下达指令即可创建任务            │
│          [前往 AI Team]          │
└──────────────────────────────────┘
```

#### 2.3.4 交互功能

| 功能 | 触发方式 | 行为 |
|------|---------|------|
| **刷新** | 点击右上角 🔄 按钮 | 重新请求 `proclawTasks.getStats()` + `proclawTasks.list()` |
| **查看详情** | 点击任务行 | 弹出 Dialog 显示任务完整信息：描述、预期输出、结果、时间线 |
| **跳转 Agent** | 点击 Agent 名称 | 跳转到 AI Team 页面并定位到该 Agent |
| **取消任务** | 待处理/进行中任务行的操作菜单 | 调用 `proclawTasks.cancel(taskId)`，刷新列表 |

### 2.4 数据流

```
DataCenterPage (tabValue === 3)
  │
  ├── useEffect → proclawTasks.getStats()
  │     └── 渲染统计卡片行
  │
  ├── useEffect → proclawTasks.list()
  │     ├── 按 agent 分组 → 渲染 Agent 分布图
  │     ├── 按日期聚合 → 渲染趋势图
  │     └── 筛选 + 分页 → 渲染任务列表
  │
  └── proclawAgents.list()
        └── 提供 Agent ID → 名称的映射表
```

**加载策略**：
- 首次渲染显示骨架屏（Skeleton）
- 数据加载失败显示错误提示 + 重试按钮
- 统计卡片先行渲染（`getStats` 轻量），图表和列表延迟不超过 500ms 渲染
