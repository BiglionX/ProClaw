## 1. 背景与目标
- **现状**：个体户/小商户很少主动盘点，期初库存不准，导致传统进销存无法正常使用（负库存报错、报表不可信）。
- **目标**：允许用户在库存数据不完美的情况下正常进行进货、销售、收银等日常操作，并通过轻量交互逐步校准库存，最终使库存数据趋向准确。

## 2. 用户角色
- 店主 / 店员（统称“用户”）

## 3. 核心功能需求

### 3.1 允许负库存销售（软约束）
- **规则**：用户销售时，若商品账面库存不足（包括为0或负），系统**允许**继续销售，不强制拦截。
- **交互**：销售完成时，在界面底部（或非模态提示）显示警告：“【商品名】账面不足，已记录为负库存。请尽快校准。”
- **记录**：负库存值必须被持久化，并在后续进货/盘点时参与冲销逻辑。

### 3.2 进货自动冲销负库存
- **规则**：当用户录入进货单（增加某商品库存）时，系统先查询该商品的当前库存（可能为负）。
  - 若当前库存 ≥ 0，正常累加进货数量。
  - 若当前库存 < 0（例如 -5），则用进货数量优先抵冲负数：  
    `新库存 = 当前库存 + 进货数量`（例：-5 + 10 = 5），并在进货单明细中标注 **“已自动冲销缺货 -5 件”**。
- **交互**：冲销提示应清晰但不过度打扰，用户可点击查看详情。

### 3.3 微盘点（零散校准）
用户可以通过简单操作（不中断业务）修正单个或少量商品的库存。

#### 3.3.1 销售后触发微盘点
- **时机**：用户销售某商品后，系统评估以下条件决定是否弹出微盘点建议（弹窗或侧边按钮）：
  - 该商品距离上次校准时间 > 3天（可配置）
  - 该商品近一周销量排名前 20%（热销品）
  - 用户当前收银动作间隔 > 3秒（非高峰忙时）
- **交互**：弹出一个极简卡片，显示商品名、当前账面库存，并提供一个数字键盘让用户输入 **实际数量**。用户确认后立即覆盖账面库存，并记录校准时间。

#### 3.3.2 进货后触发微盘点
- **时机**：用户完成进货单录入后，自动跳转（或弹窗）提示：“刚刚进了【商品名】，现在货架上实际有多少？” 用户可以输入实际数量（系统默认值为原账面库存 + 进货数量）。
- **作用**：一次进货动作同时完成进货记账 + 货架校准。

#### 3.3.3 手动微盘点入口
- **位置**：商品详情页、库存列表页提供“校准库存”按钮，点击后弹出数字键盘，输入实际数量即更新。

### 3.4 库存置信度模型（后端计算，UI展示）
- **定义**：每个商品有一个置信度评分（高/中/低），影响提醒策略和报表展示。
- **计算公式（简化）**：
  - 初始 = 低
  - 最近7天内有微盘点 → 高
  - 最近7天内有进货但无微盘点 → 中
  - 超过15天无任何校准动作 → 低
  - 存在负库存且未冲销 → 低
- **展示**：在商品列表或库存页面用图标显示（🟢🟡🔴），销售时若为低置信度，可额外提示“建议先核对”。

### 3.5 AI自动提醒（可配置）
- **提醒类型**：
  1. **负库存持续提醒**：某商品负库存超过 3 天且未冲销 → 每日首次启动软件时弹窗提醒（用户可点击“去校准”或“忽略今天”）。
  2. **长期未校准提醒**：置信度为低且超过 15 天未校准 → 在桌面端右下角/移动端通知栏显示非模态消息，点击可进入微盘点。
  3. **高销量低置信度提醒**：近7天销量排名前10%但置信度为低 → 主动推送“【商品名】卖得很快，库存可能不准，花5秒核实一下”。
- **防骚扰机制**：用户连续忽略同一提醒3次，则降低该商品提醒频率（7天内不再提醒同类）。

### 3.6 负库存“老化”处理
- **规则**：若某商品负库存状态持续超过 **14天** 且没有任何进货/校准动作：
  - 系统自动生成一个“待处理任务”，并高亮显示在仪表盘。
  - 用户必须手动处理（选择“强制清零”或“补录入库”）才能继续对该商品进行销售。
- **交互**：处理时弹窗解释原因，提供两个选项：
  - “确认报废并清零库存”
  - “补录一批入库（用户输入数量）”

## 4. 报表与数据一致性要求
- **利润报表**：当存在负库存时，成本核算采用 **移动加权平均**（基于历史进货），负库存部分不重复计算毛利，并在报表底部添加备注：“库存不准可能影响利润准确性，建议先校准”。
- **库存流水**：所有校准动作（包括微盘点、冲销、负数强制清零）必须记录为“库存调整单”，可追溯。

## 5. 非功能需求
- **响应速度**：销售完成后的微盘点弹窗延迟 < 200ms。
- **离线支持**：移动端在无网络时仍可记录销售和微盘点，联网后同步。
- **多端一致性**：桌面端、移动端的库存数据实时同步（WebSocket 或轮询）。

## 6. 验收标准（示例）
- [ ] 能够销售账面为0的商品，生成负库存记录，不报错。
- [ ] 进货时若商品为负库存，自动冲销并在进货单上显示冲销信息。
- [ ] 销售高频商品后3秒内出现微盘点建议弹窗。
- [ ] 手动校准后置信度立即变为“高”，图标变化。
- [ ] 负库存持续14天后，系统禁止销售并提示处理。
- [ ] AI提醒可以基于条件触发，且用户忽略3次后不再骚扰。
