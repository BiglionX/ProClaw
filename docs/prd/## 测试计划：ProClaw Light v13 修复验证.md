# ProClaw Light v13 修复验证测试计划

## 测试目标

验证 ProClaw Light 版 v13.0 所有修复的正确性，确保：
1. P0 功能断裂修复完全生效
2. P1 安全性修复有效防止已知风险
3. P2 代码质量修复不引入新问题
4. Light 版可作为 Plus 版的稳定子集运行

## 测试范围

### 版本配置
- **Plus 版**：`cargo check --features inventory`
- **Light 版**：`cargo check --no-default-features --features "custom-protocol,light"`

### 修复清单
- P0-1: Light AI 助手商品库数据断路
- P0-2: Light 版安装向导配置丢失
- P0-3: 知识库中文文本乱码
- P1-1: Rust `light` feature flag
- P1-2: 模拟账号安全加固
- P1-3: Agent 沙箱路径穿越防护
- P1-4: 插件加载路径白名单
- P2-1: AppMode 类型修正
- P2-2: min_stock 语义修正
- P2-3: 消息 ID 碰撞修复
- P2-4: HTTPS 强制
- P2-5: 错误信息保留
- P2-6: LLM 调用防护

## 风险评估

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|----------|
| Light 编译配置遗漏 | 高 | 中 | 已完成 feature flag 验证 |
| 模拟账号生产泄漏 | 高 | 低 | 环境变量隔离，无变量自动失效 |
| 插件白名单绕过 | 高 | 低 | 路径规范化 + 白名单双重检查 |
| Plus 版回归 | 高 | 中 | 必须执行 Test-04 |
| 数据迁移不兼容 | 中 | 高 | 建议执行 Test-14 |
| Rust 端错误传播 | 中 | 中 | 需验证所有调用方 |
