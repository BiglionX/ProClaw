### 1. 背景与现状分析

#### 1.1 当前 UI 审计结论

经过对桌面端全部核心页面的逐项审查，当前 UI 存在以下系统性缺陷：

| 问题领域 | 严重度 | 具体表现 |
|----------|--------|----------|
| **缺乏 AI 感** | 🔴 高 | 整体风格是传统的 MUI 默认 CRUD 应用，用户难以感知这是一款 AI 驱动的产品 |
| **设计语言碎片化** | 🔴 高 | 侧边栏深色 + 内容区浅色 + 各类组件拼凑感强，没有统一的设计体系 |
| **数据呈现无活力** | 🟡 中 | 图表用 Recharts 默认样式，卡片无动效，数字缺少"动起来"的生命感 |
| **侧边栏单调** | 🟡 中 | 纯文字列表，无品牌 Logo，无用户信息，无分组标签，无折叠能力 |
| **顶栏信息密度低** | 🟡 中 | 仅有 Logo + 用户头像，缺少面包屑、全局搜索、通知中心 |
| **页面动线薄弱** | 🟡 中 | 所有页面从标题开始 → 操作区 → 表格/图表结束，千篇一律 |
| **AI 能力不可见** | 🟠 极高 | 除了右下角的秘书头像，全系统界面看不出任何 AI 元素——AI 知识库、AI 团队、AI 客服等功能在自己的页面里，但在主界面没有任何 AI 感知 |

#### 1.2 核心理念

> **ProClaw 不只是一个"进销存工具"，它是一个 AI 驱动的经营智能体系统。UI 必须让用户在第一眼就感知到这一点。**

---

### 2. 设计目标

1. **AI-Native 第一印象**：用户打开 ProClaw 的瞬间，就能感受到 AI 的存在——不是浮于表面的炫技，而是嵌入工作流的智能
2. **专业可信赖**：拒绝花哨的科幻感，用克制、精准、温暖的 AI 视觉语言传达专业度
3. **数据会呼吸**：卡片、图表、数字应具备微妙的动效反馈，让数据"活起来"
4. **信息架构清晰**：通过分组、渐进式披露、视觉层级，让复杂的 AI 功能体系一目了然
5. **统一设计系统**：建立专属的 Design Token，告别 MUI 默认风格

- 保持当前横向可滑动的正方形卡片布局 ✅（已实现）
- 每张卡片增加微渐变背景（根据颜色生成 `linear-gradient` 浅变）

### 5. 交互细节升级（全局）

#### 5.1 页面切换动效

- **路由切换**：使用 200ms 的淡入淡出（`opacity` + `translateY(4px)` → `0`）
- **Tab 切换**：使用 MUI Tab 的 inkBar + 自定义 300ms 过渡
- **数据面板展开**：使用 CSS `transition` + `max-height` 实现顺滑展开

#### 5.2 数据加载体验

| 场景 | 当前 | 改进 |
|------|------|------|
| **首次加载** | `CircularProgress` 旋转 | 骨架屏（Skeleton）——表格骨架 + 卡片骨架 |
| **刷新数据** | 同上 + 页面闪烁 | 顶部 **细进度条**（NProgress 风格），数据就地刷新不跳动 |
| **空状态** | 灰色文字居中 | 空状态插图 + AI 引导语 + 快捷操作按钮 |

#### 5.3 微交互（Micro-interactions）

| 交互 | 实现 |
|------|------|
| **数字跳动** | 加载数据时，指标数字从 0 快速计数到目标值（使用 `countup.js` 或 CSS `@keyframes`） |
| **卡片 hover** | 上浮 2-4px + 阴影加深 + 边框色渐变到品牌色 |
| **按钮点击** | 120ms 的 `scale(0.96)` + 回弹动画 |
| **通知徽章** | 出现时带有 `scale(0) → scale(1.2) → scale(1)` 的弹性动画 |
| **AI 思考中** | Tab 标签栏切换时，文字短暂显示"🤔 AI 分析中..." + 三个点依次闪烁 |

#### 5.4 AI 悬浮秘书增强

当前 FloatingAgentChat 已经设计得不错（右键菜单、BAP、碰壁话术等），建议：
- **消息气泡增加 AI 标识**：每条 AI 回复前增加微小的品牌色小点
- **打字机效果**：流式输出时使用逐字打字动画
- **集成到更多页面**：在数据中心、AI 团队等页面的右侧固定位置嵌入迷你秘书面板（非浮动）

### 8. 技术实现建议

#### 8.1 动效库选择

- **数字计数**：使用 `countup.js`（或 `react-countup`）
- **骨架屏**：使用 `@mui/material/Skeleton`
- **页面过渡**：使用 `framer-motion` 的 `AnimatePresence`

### 9. 风险与应对

| 风险 | 应对 |
|------|------|
| **动画性能** | 使用 `will-change` 和 `transform` 硬件加速，避免在表格行数多时做复杂动效 |
| **AI 洞察准确性** | 先用规则引擎生成（如"库存 < 10 → 提示补货"），后续接入 LLM 生成自然语言 |
| **开发周期长** | 按 P0 → P1 → P2 分阶段交付，P0 完成后即可发布，后续持续迭代 |
| **与插件体系兼容** | 全局主题升级在基础层实现，插件页面通过 MUI Theme 自动继承新样式 |
