# ProClaw 项目定位说明

> **重要**：本文档明确 ProClaw 项目的核心定位，避免与类似项目混淆。

## 🎯 项目定位

**ProClaw** 是一个**开源的AI驱动商户经营操作系统**（Open-Source AI-Powered Business OS）。

### 产品类型
- **桌面应用**：基于 Tauri 2.0 构建的跨平台桌面应用
- **开源项目**：GPL-3.0 许可证，代码完全开放
- **AI原生**：深度集成大语言模型和AI智能体
- **本地优先**：数据默认存储在本地，用户完全掌控

### 目标用户
1. **中小商户** - 便利店、超市、专卖店、连锁店等
2. **开发者** - 希望扩展功能或自建部署的技术人员
3. **企业IT管理员** - 需要私有化部署和数据安全的企业

## 🔑 核心功能重点

ProClaw 的核心不是传统的进销存软件，而是一个**AI赋能的业务操作系统**，重点在于：

### 1. 用户管理系统
- 注册/登录/权限控制
- 多租户支持
- 角色权限管理（店主/仓管/财务）
- 会话管理和安全认证

### 2. 大模型调用能力
- AI经营智能体（自然语言交互）
- Dify工作流引擎集成
- 智能数据分析（销量预测、库存预警、利润分析）
- 上下文记忆和多轮对话
- 自定义Prompt模板

### 3. Token管理
- API密钥管理（支持多个LLM提供商）
- 用量统计和监控
- 成本控制和建议
- 配额管理和限制
- 费用分析和优化建议

### 4. 外部资源桥接
- RESTful API集成
- Webhook支持
- 第三方服务连接（支付、物流、ERP、电商等）
- 自定义连接器开发
- API网关和路由管理

### 5. 技能生态
- 可扩展的插件系统
- 技能商店（财务管理、会员管理、电商对接等）
- 开发者SDK和文档
- 收益分成机制
- 社区贡献和审核

## 💾 部署模式

### 🖥️ 本地优先（默认 - 免费）

**架构**：
```
用户设备
  ├── Tauri Desktop App
  ├── SQLite (SQLCipher加密)
  ├── 本地文件系统
  └── [可选] Supabase 云端同步
```

**特点**：
- ✅ 完全免费，开源代码
- ✅ 数据100%本地存储，隐私安全
- ✅ 离线可用，不依赖网络
- ✅ 完全掌控，无供应商锁定
- ✅ 可自行定制和扩展
- ⚠️ 需要自行维护和数据备份
- ⚠️ 多设备同步需手动配置

**适合人群**：
- 注重数据主权的用户
- 有技术能力的用户
- 追求完全控制的用户
- 预算有限的个人或小团队

### ☁️ 云托管（可选 - 付费订阅）

**架构**：
```
用户设备
  ├── Tauri Desktop App
  └── Supabase Backend (官方托管)
        ├── PostgreSQL
        ├── Auth + RLS
        ├── Realtime Sync
        └── Edge Functions
```

**特点**：
- ✅ 自动数据同步和备份
- ✅ 多设备实时协作
- ✅ 无需自行维护基础设施
- ✅ 专业技术支持和SLA保障
- ✅ 定期更新和新功能
- 💰 需要付费订阅

**定价**：
- 基础版：¥99/月
- 专业版：¥299/月
- 企业版：联系销售定制

**适合人群**：
- 追求便捷的用户
- 需要团队协作的企业
- 不想自行维护的用户
- 需要专业技术支持的用戶

## ❌ 不是什么

为了避免混淆，明确 ProClaw **不是**：

1. ❌ **不是维修服务管理平台**（如 ProCyc/FixCycle）
   - 不涉及设备维修、配件商城、售后服务
   
2. ❌ **不是纯SaaS应用**
   - 默认是本地桌面应用，云托管是可选服务
   
3. ❌ **不是传统进销存软件**
   - 重点是AI能力和扩展性，不只是简单的库存管理
   
4. ❌ **不是企业级ERP**
   - 面向中小商户，轻量级、易用性强
