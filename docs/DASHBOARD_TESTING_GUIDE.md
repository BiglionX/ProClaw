## 测试数据准备

为了看到完整的仪表盘效果，建议在数据库中准备以下测试数据：

### 1. 创建产品

至少创建 10-20 个产品，包括：
- 不同库存水平的产品（正常、低库存、零库存）
- 不同分类的产品
- 不同价格的产品

### 2. 创建库存交易

创建一些入库和出库记录：
```sql
-- 示例：创建出库交易
INSERT INTO inventory_transactions (id, product_id, transaction_type, quantity, created_at)
VALUES ('uuid-1', 'product-id-1', 'outbound', 10, datetime('now'));

-- 示例：创建入库交易
INSERT INTO inventory_transactions (id, product_id, transaction_type, quantity, created_at)
VALUES ('uuid-2', 'product-id-2', 'inbound', 20, datetime('now'));
```

### 3. 创建销售订单

创建一些销售订单以生成本月销售数据：
```sql
INSERT INTO sales_orders (id, so_number, customer_id, order_date, total_amount, status)
VALUES ('uuid-3', 'SO-TEST001', 'customer-id-1', date('now'), 5000.00, 'confirmed');
```

### 4. 创建采购订单

创建一些采购订单以生成应付账款：
```sql
INSERT INTO purchase_orders (id, po_number, supplier_id, order_date, total_amount, status)
VALUES ('uuid-4', 'PO-TEST001', 'supplier-id-1', date('now'), 3000.00, 'confirmed');
```

## 常见问题

### Q: 所有数据都显示为 0？
A: 这是因为数据库中还没有相关记录。请按照上面的"测试数据准备"部分添加测试数据。

### Q: 图表不显示？
A: 确保数据库中有足够的历史数据（至少 7 天的交易记录）。

### Q: 加载很慢？
A: 首次加载需要并行请求多个 API，可能需要几秒钟。后续可以通过缓存优化。

### Q: 出现错误提示？
A: 检查：
1. 后端 Rust 服务是否正常运行
2. 数据库连接是否正常
3. 浏览器控制台是否有错误信息

## 性能测试

### 加载时间测试

使用浏览器开发者工具的 Network 面板检查：
- 总加载时间应 < 3 秒
- 单个 API 请求应 < 500ms
- 并行请求数量 = 5

### 内存使用

使用浏览器开发者工具的 Performance 面板检查：
- 初始内存占用应 < 50MB
- 无内存泄漏（多次刷新后内存稳定）
