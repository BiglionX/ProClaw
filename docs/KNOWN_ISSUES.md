> 最后更新: 2026-04-12

本文档列出当前 Beta 版本的已知问题和限制。我们会在每个版本中逐步修复这些问题。

---

## 🔴 严重问题 (P0)

目前没有阻塞使用的严重问题。

---

## 🟡 重要问题 (P1)

### 1. AI 功能为简化版

**问题描述**:
当前的"AI 智能体"使用规则匹配（正则表达式），而非真正的 AI 模型。

**影响**:

- 只能识别预定义的指令模式
- 无法理解复杂的自然语言
- 不支持多轮对话上下文

**临时解决方案**:
使用快捷操作面板或直接导航到对应页面。

**预计修复**: v0.5.0（集成 Dify/Qwen）

---

### 2. 云端同步未完全实现

**问题描述**:
Supabase 同步引擎的部分功能为模拟实现（TODO 标记）。

**具体表现**:

- `sync_to_supabase()` 函数返回模拟数据
- 实际未调用 Supabase REST API
- 冲突解决策略未完整测试

**影响**:

- 演示模式下无影响（纯本地）
- 配置 Supabase 后，数据不会真正同步到云端

**临时解决方案**:
使用演示模式，或等待后续版本修复。

**预计修复**: v0.2.0

---

### 3. 图片存储效率低

**问题描述**:
产品图片以 Base64 格式存储在 SQLite 数据库中。

**影响**:

- 数据库文件体积快速增长
- 大量图片时查询性能下降
- 不适合存储高分辨率图片

**临时解决方案**:

- 上传前压缩图片（建议 < 500KB）
- 定期清理不需要的产品

**预计修复**: v1.0.0（改为文件系统存储 + 数据库存路径）

---

## 🟢 轻微问题 (P2)

### 4. 首次启动较慢

**问题描述**:
首次启动应用需要 3-5 秒初始化数据库和加载资源。

**影响**:
用户体验稍差，但后续启动会更快。

**临时解决方案**:
耐心等待首次启动完成。

**预计修复**: v0.2.0（优化初始化流程）

---

### 5. 列表未使用虚拟滚动

**问题描述**:
产品和交易记录列表使用普通渲染，大数据量时可能卡顿。

**影响**:

- 1000+ 条记录时滚动不够流畅
- 内存占用较高

**临时解决方案**:
使用筛选功能减少显示的数据量。

**预计修复**: v0.2.0（集成 react-window）

---

### 6. 同步状态显示不准确

**问题描述**:
`getSyncStatus()` 中的 `last_sync_time` 从 localStorage 读取，可能与实际不符。

**影响**:
用户看到的最后同步时间可能不准确。

**临时解决方案**:
手动刷新页面或重新触发同步。

**预计修复**: v0.2.0

---

### 7. macOS 应用未签名

**问题描述**:
macOS 安装包未进行 Apple Developer 签名和公证。

**影响**:

- 首次启动需要手动允许
- Gatekeeper 警告
- Catalina+ 系统额外步骤

**临时解决方案**:
按照安装指南中的步骤绕过 Gatekeeper。

**预计修复**: v0.2.0（需要 Apple Developer 账号）

---

### 8. Windows SmartScreen 警告

**问题描述**:
Windows 显示"未知发布者"警告。

**影响**:
用户需要额外点击才能安装。

**临时解决方案**:
点击"更多信息" → "仍要运行"。

**预计修复**: v1.0.0（需要 EV 代码签名证书）

---

### 9. 缺少自动更新机制

**问题描述**:
新版本发布后，用户需要手动下载并安装。

**影响**:
用户体验不便，升级率低。

**临时解决方案**:
关注 GitHub Releases 页面或订阅邮件通知。

**预计修复**: v0.2.0（集成 Tauri Updater）

---

### 10. 部分快捷操作未实现

**问题描述**:
QuickActions 组件中的"创建订单"和"生成报表"按钮仅显示提示。

**影响**:
功能不完整，但不影响核心使用。

**临时解决方案**:
这些功能将在 v1.0 正式版提供。

**预计修复**: v1.0.0

## 🔍 如何报告新问题

如果您发现了未列出的问题:

### 步骤 1: 检查是否已报告

浏览 [GitHub Issues](https://github.com/your-org/proclaw-desktop/issues) 确认问题未被报告过。

### 步骤 2: 收集信息

- 操作系统和版本
- ProClaw 版本号
- 复现步骤
- 预期行为 vs 实际行为
- 截图或录屏（如有）
- 日志文件位置:
  - Windows: `%APPDATA%\com.proclaw.desktop\logs\`
  - macOS: `~/Library/Logs/com.proclaw.desktop/`
  - Linux: `~/.local/share/com.proclaw.desktop/logs/`

### 步骤 3: 提交 Issue

访问 [GitHub Issues](https://github.com/your-org/proclaw-desktop/issues/new) 创建新问题。

**Issue 模板**:

```markdown
**问题描述**:
[清晰简洁地描述问题]

**复现步骤**:

1.
2.
3.

**预期行为**:
[应该发生什么]

**实际行为**:
[实际发生了什么]

**环境信息**:

- 操作系统: [例如 Windows 11 22H2]
- ProClaw 版本: [例如 v0.1.0-beta]
- 硬件配置: [可选]

**日志文件**:
[附上相关日志]

**截图**:
[如有]
```
