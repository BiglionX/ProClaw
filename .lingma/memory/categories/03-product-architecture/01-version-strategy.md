# 多版本产品线定义与发布策略

> 创建时间：2026-06-08  
> 最后更新：2026-06-08  
> 关键词：产品版本、发布策略、版本管理、ProClaw Lite、Plus

## 产品版本体系

### 1. ProClaw Light（轻量版）
- **定位**：面向小型商户的轻量解决方案
- **特性**：
  - 桌面端应用（Tauri）
  - 基础进销存功能
  - 简洁的UI界面
- **发布节奏**：稳定版季度更新

### 2. ProClaw Cloud-Store（云端版）
- **定位**：SaaS化的云端管理系统
- **特性**：
  - Next.js + Supabase
  - 多租户支持
  - AI能力集成（DeepSeek）
- **发布节奏**：持续迭代

### 3. ProClaw Mobile（移动版）
- **定位**：移动办公场景
- **技术栈**：React Native (Expo)
- **特性**：
  - 原生移动体验
  - 离线数据同步
  - 消息推送通知
- **发布节奏**：App Store/Play Store 独立更新

### 4. ProClaw Plus（增强版）
- **定位**：面向中大型企业的增强版
- **特性**：
  - 完整AI能力
  - 高级分析功能
  - 定制化服务
- **发布节奏**：年度大版本

## 版本号规范

采用语义化版本（Semantic Versioning）：
- **主版本号**：架构变更、破坏性更新
- **次版本号**：新增功能、向后兼容
- **修订号**：Bug修复、文档更新

格式：`MAJOR.MINOR.PATCH`（如：1.2.3）

## 发布策略

### 发布流程
1. 功能开发完成
2. 代码审查（Code Review）
3. 测试验证（E2E测试）
4. 文档更新
5. 版本打标签
6. 发布上线

### 回滚机制
- 云端版本：蓝绿部署
- 桌面端：自动更新 + 回滚支持
- 移动端：App Store审核后生效

## 版本兼容性

| 版本组合 | 兼容性说明 |
|----------|------------|
| Light + Mobile | 数据格式兼容 |
| Cloud-Store + Mobile | API接口兼容 |
| Plus → Light | 功能子集关系 |
