# 简单商品库自动编码功能

## 📋 功能概述

为ProClaw桌面端的**简单商品库**添加了**SKU自动编码**功能，让商品录入更高效、更规范。

### 核心特性

✅ **自动生成SKU** - 新建商品时自动生成唯一编码  
✅ **智能格式** - `SKU-{YYYYMMDD}-{随机4位}`  
✅ **可手动修改** - 生成的编码可以自由编辑  
✅ **一键刷新** - 点击"重新生成"按钮获取新编码  
✅ **向后兼容** - 支持手动输入自定义SKU  
✅ **双重保障** - 前端生成 + 后端验证  

---

## 🎯 SKU编码规则

### 编码格式

```
SKU-{YYYYMMDD}-{随机4位}
```

**示例**：
- `SKU-20260415-A3F7`
- `SKU-20260415-B8K2`
- `SKU-20260415-M9X1`

### 编码组成

| 部分 | 说明 | 示例 |
|------|------|------|
| 前缀 | 固定为"SKU-" | `SKU-` |
| 日期 | 年月日（8位） | `20260415` |
| 随机码 | 4位大写字母+数字 | `A3F7` |

### 唯一性保证

- ✅ 日期部分确保不同天的编码不重复
- ✅ 随机码（36进制）提供 36^4 = 1,679,616 种组合
- ✅ 同一天内重复概率极低（约1/160万）
- ⚠️ 如需强唯一性，建议后端添加唯一索引检查

## 🔧 技术实现

### 1. 编码生成函数

**文件**: `src/lib/productService.ts`

```typescript
/**
 * 生成简单商品库的SKU编码（兼容旧版）
 * 格式: SKU-{YYYYMMDD}-{随机4位}
 * 示例: SKU-20260415-A3F7
 */
export function generateSimpleSKU(): string {
  const now = new Date();
  const dateStr = now.toISOString().slice(0, 10).replace(/-/g, '');
  const randomStr = Math.random().toString(36).substring(2, 6).toUpperCase();
  return `SKU-${dateStr}-${randomStr}`;
}
```

**实现细节**：
1. 获取当前日期并格式化为`YYYYMMDD`
2. 使用`Math.random().toString(36)`生成36进制随机字符串
3. 截取第2-6位（跳过"0."），得到4位随机码
4. 转换为大写并拼接完整SKU

### 2. 自动填充逻辑

**文件**: `src/pages/ProductsPage.tsx`

```typescript
// 打开新建对话框时自动生成SKU
const handleOpenDialog = (product?: ProductType) => {
  if (product) {
    // 编辑模式：使用现有SKU
    setFormData({ sku: product.sku, ... });
  } else {
    // 新建模式：自动生成SKU
    const autoSKU = generateSimpleSKU();
    setFormData({ sku: autoSKU, ... });
  }
};
```

### 3. 后端自动补全

**文件**: `src/lib/productService.ts`

```typescript
export async function createProduct(input: CreateProductInput): Promise<Product> {
  // 如果未提供SKU，自动生成
  const finalInput = {
    ...input,
    sku: input.sku || generateSimpleSKU(),
  };
  
  // 继续创建流程...
}
```

**双重保障**：
- 前端：UI层自动生成，用户可见可改
- 后端：API层再次检查，确保一定有SKU

### 4. UI交互设计

```tsx
<Box sx={{ display: 'flex', gap: 1 }}>
  <TextField
    label="SKU（自动生成，可修改）"
    value={formData.sku}
    onChange={e => setFormData({ ...formData, sku: e.target.value })}
    helperText="格式: SKU-日期-随机码"
  />
  {!editingProduct && (
    <Button onClick={() => setFormData({ ...formData, sku: generateSimpleSKU() })}>
      重新生成
    </Button>
  )}
</Box>
```

**UI特点**：
- 清晰的标签提示"自动生成，可修改"
- HelperText显示编码格式
- 仅在新建模式显示"重新生成"按钮
- 编辑模式不允许重新生成（避免改变已有编码）

**优势**：灵活性高，可使用业务相关编码

### 场景3: 对自动编码不满意

1. 点击"添加产品"
2. 看到自动生成的SKU
3. 点击"重新生成"按钮

5. 可多次点击直到满意

**优势**：提供更多选择

### 场景4: 批量录入相似商品

1. 第一个商品使用自动编码
2. 第二个商品点击"重新生成"
3. 或者手动修改后缀（如：`SKU-20260415-A3F8`）

**优势**：保持编码规律性

## ⚠️ 注意事项

### 1. 编辑模式限制

**问题**：编辑商品时为什么不显示"重新生成"按钮？

**原因**：
- 已有商品可能被其他数据引用（订单、库存等）
- 改变SKU可能导致关联数据断裂
- 保持数据一致性

**解决方案**：
- 如确实需要修改，可直接在输入框中编辑
- 系统不会阻止手动修改

### 2. 并发冲突

**问题**：两个用户同时创建商品，会生成相同SKU吗？

**分析**：
- 概率极低（同一天内1/160万）
- 但如果发生，会导致数据库唯一约束冲突

**解决方案**：
```sql
-- 建议在数据库添加唯一索引
CREATE UNIQUE INDEX idx_products_sku ON products(sku);
```

```typescript
// 后端捕获冲突并重试
try {
  await db.insert(...);
} catch (err) {
  if (err.code === 'UNIQUE_CONSTRAINT') {
    // 重新生成SKU并重试
    const newSku = generateSimpleSKU();
    await db.insert({ ...data, sku: newSku });
  }
}
```

### 3. 时区问题

**问题**：日期是基于本地时区还是UTC？

**当前实现**：
```typescript
const now = new Date();
const dateStr = now.toISOString().slice(0, 10); // UTC时间
```

使用的是**UTC时间**，可能导致：
- 中国用户晚上23点创建的商品，日期可能是明天

**改进方案**：
```typescript
// 使用本地时间
const now = new Date();
const year = now.getFullYear();
const month = String(now.getMonth() + 1).padStart(2, '0');
const day = String(now.getDate()).padStart(2, '0');
const dateStr = `${year}${month}${day}`;
```
