## 一、项目背景

### 1.1 市场机会

短视频已成为中小商家获客的核心渠道，但绝大多数个体户（餐饮、美发、零售等）面临三大痛点：

1. **不会拍**：没有拍摄经验，不知道如何分镜、如何展示产品。

2. **不会写**：缺乏文案能力，生成的视频千篇一律，没有吸引力。

3. **没有IP**：无法形成个人品牌，视频缺乏辨识度。

与此同时，AIGC技术（文案生成、音色克隆、自动混剪）已趋于成熟，具备产品化的条件。

### 1.2 产品定位

**ProClips（拍可丽）** 是基于 Proclaw 移动端的衍生独立 App，面向中小商家和达人，提供 AI 视频营销与分销能力。

- **双角色一体化设计**：商家（生成营销视频）+ 达人（分销赚佣），两个角色一次性完成

- **不分阶段交付**：很多商家本身就是达人，可做异业推荐，因此商家与达人两个角色一次性完成，不再分阶段

- **AI 智能匹配**：通过 AI 智能匹配实现商家与达人的双向连接——商家主动邀请达人，达人主动申请推广

### 1.3 与 Proclaw 的关系

ProClips 是 **基于 Proclaw 移动端的衍生产品**，复用 Proclaw 的核心技术栈（认证、数据库、Agent 生态、上传组件），但作为 **独立 App** 发布，拥有独立的 UI 设计、功能集和 applicationId，可与 ProClaw Mobile 同时安装在同一设备上。

- **用户**：独立 App 注册登录（复用 Proclaw 账号体系）

- **交互**：独立的 ProClips App UI，围绕"与广告主助手对话"展开

- **后端**：复用 Proclaw Skill API 和混剪服务，与 Proclaw 主后端协同

- **数据**：使用 Proclaw 现有数据库，新增业务表（前缀 `video_`）

### 3.2 双角色体系与魔幻变身

ProClips 采用双角色一体化设计，一个账号可同时拥有商家和达人两种身份，无需重复注册。

**3.2.1 商家角色**

- 生成营销视频（拍摄引导、AI 文案、音色克隆、混剪）

- 设置推广激励（佣金、奖励等，详见 3.3）

- 主动邀请达人推广（AI 智能匹配，详见 3.4）

- 管理视频库与分销数据

**3.2.2 达人角色**

- 浏览商家素材库（按行业/佣金/热度筛选）

- AI 选品（根据偏好智能推荐视频，详见 3.4）

- 申请推广商家视频

- 赚取佣金与奖励

**3.2.3 角色切换与魔幻变身**

- 入口："我的"页面角色切换入口

- 切换动画：3 秒魔幻变身动画（光环展开 + 粒子飞散 + 图标 morph）

- 一个账号可同时拥有两种身份，支持异业推荐场景（例：餐饮商家同时是美食达人，可推荐其他行业商品）

### 3.3 推广激励方式设置（商家侧）

商家在公开视频到素材库时，可设置推广激励方式。5 种方式可叠加组合：

| 序号 | 激励方式 | 说明 | 示例 |

|------|----------|------|------|

| 1 | 比例佣金（CPS） | 订单金额百分比，可设最低保底 | 15% + 最低保底 ¥10/单 |

| 2 | 固定佣金 | 每完成一单固定金额 | ¥20/单 |

| 3 | 阶梯佣金 | 按订单量分档累进 | 1-10 单 10%，11-50 单 15%，50+ 单 20% |

| 4 | 播放奖励（CPM） | 每千次播放固定金额，可设封顶 | ¥10/千次，封顶 ¥500 |

| 5 | 达标奖金 | 达到订单数或播放数后一次性奖金 | 满 100 单奖励 ¥500 |

商家可在视频库中对每条视频独立配置激励方案；达人侧在素材库中可直观看到每条视频的激励组合，便于选品决策。

### 3.8 第三方平台接入与多平台分发

**3.8.1 平台账号管理**

入口："我的" → "第三方平台"，徽章显示已绑定数量（如 4/6）。

支持 6 个主流短视频/图文平台：

| 平台 | 主色 | 授权方式 |
|------|------|----------|
| 抖音 | 黑 | 抖音开放平台 OAuth 2.0 |
| 快手 | 橙 | 快手开放平台 OAuth 2.0 |
| 小红书 | 红 | 小红书开放平台 OAuth 2.0 |
| 视频号 | 绿 | 微信开放平台（需企业资质） |
| B 站 | 蓝 | 哔哩哔哩开放平台 OAuth 2.0 |
| 微博 | 橙红 | 微博开放平台 OAuth 2.0 |

已绑定平台卡片展示：账号头像、昵称、粉丝数、今日播放/点赞/评论、最近同步时间，提供"同步数据"与"解绑"操作。顶部汇总卡显示已绑定数量与覆盖粉丝总数。

**3.8.2 OAuth 授权流程（3 步状态机）**

点击未绑定平台的"绑定账号"进入授权页，分 3 步：

- **Step 1 授权确认**：平台主色 hero + 5 项权限勾选（发布视频 / 读取数据 / 管理评论 / 获取粉丝信息 / 管理消息[可选]）+ "同意授权"
- **Step 2 平台登录**：手机号 + 验证码（"获取验证码"）+ "登录并授权"
- **Step 3 绑定成功**：绿色大勾动画 + 账号信息卡（头像/昵称/粉丝数）+ "完成"返回平台列表并刷新

授权后 access_token 与 refresh_token 持久化存储，支持自动续期；用户可随时解绑。

**3.8.3 多平台一键分发**

入口：视频库每个视频卡片的"📤 一键分发"按钮。

交互流程：
1. 点击后底部弹出平台选择面板（bottom-sheet）
2. 列出已绑定平台，多选 Checkbox 默认全选
3. 每个平台显示：图标 / 粉丝数 / 预计曝光（粉丝数 × 8%）
4. 点击"一键分发到 N 个平台"进入发布状态页

**3.8.4 发布状态追踪**

发布状态页（publish-status）实时展示各平台发布进度：

| 状态 | 展示 |
|------|------|
| 发布中 | loading 动画 + 进度百分比（如 65%） |
| 成功 | ✓ + 播放数 / 点赞数 / 平台链接 |
| 失败 | ✗ + 失败原因 + "重试"按钮 |

底部汇总卡显示总曝光与总互动，数据持续更新。

**3.8.5 技术方案**

- **授权**：各平台统一走 OAuth 2.0 授权码模式，Token 表存储 `access_token` / `refresh_token` / `expires_at`，定时任务自动续期
- **发布**：各平台开放 API 上传视频（title/cover/video_url），异步轮询发布结果
- **数据回传**：定时任务（每小时）拉取各平台视频数据 API，写入 `video_publish_stats` 表
- **解绑**：调用平台 OAuth 撤销接口 + 清除本地 Token
- **降级**：某平台 API 不可用时，发布状态标记失败并支持重试，不影响其他平台

**3.10.2 System Prompt（核心）**

> 你是 ProClips 广告主助手，专门帮助中小商家制作短视频营销内容。你的工作流程严格按照以下顺序执行，每步完成后才能进入下一步：

>

> 1. **首次对话**：询问商家的店铺名称和行业，调用 `save_merchant_profile` 保存。如果用户已有记录，直接调用 `get_merchant_profile` 获取。

> 2. **模板推荐**：根据行业调用 `get_templates`，展示 3-5 个适合的模板，引导商家选择。

> 3. **拍摄引导**：根据模板的分镜列表，逐镜头指导拍摄。每段素材拍摄完成后：

>    - 调用 `generate_upload_url` 获取上传链接

>    - 提供清晰的上传指引（按钮或文字）

>    - 用户上传后调用 `confirm_scene_upload` 确认

>    - 所有镜头完成后自动进入下一步

> 4. **商品信息收集**：询问商品名称、核心卖点（1-2 个）、优惠信息（如有）。

> 5. **文案生成**：调用 `generate_script` 生成 3 个版本的文案，让用户选择或修改。

> 6. **音色处理**：

>    - 如果用户未录制音色，引导录制 20 秒标准文本，调用 `record_voice_sample`

>    - 如果已有音色，询问是否使用或重新录制

> 7. **触发混剪**：调用 `submit_mix_task` 提交任务，告知用户预计等待 3-5 分钟。

> 8. **任务跟踪**：主动轮询 `get_task_status`，完成后立即通知用户。

> 9. **视频库管理**：成品展示后，提供三个选项：

>    - 下载到本地（调用 `get_video_download_url`）

>    - 分享到朋友圈/微信群（调用 `get_share_info`）

>    - 公开到达人库（调用 `set_video_public`）

> 10. **数据汇报**：每周主动调用 `get_merchant_stats` 向用户汇报视频传播效果。

>

> **语气要求**：亲切、专业，像私人助理。每步操作给出清晰的指引。遇到错误时给出友好的重试建议。主动提醒用户未完成的步骤。

### 3.12 混剪服务需求

混剪服务是独立的微服务，与 Proclaw 主后端通过 HTTP 通信。

**3.12.1 输入参数**

```json

{

  "task_id": "task_001",

  "scene_files": ["url1", "url2", "url3"],

  "script": "欢迎来到张三火锅店...",

  "voice_id": "voice_001",

  "logo_url": "[https://oss.../logo.png](https://oss.../logo.png)",

  "bgm_id": "bgm_001",

  "template_id": "tpl_001"

}

```

**3.12.2 工作流（FFmpeg）**

1. 下载所有素材到本地临时目录

2. 按顺序拼接所有片段

3. 调用 TTS 服务（阿里云/微软）生成配音音频

4. 叠加配音到视频轨（降低原音音量至 20%）

5. 叠加 Logo 到右上角

6. 添加背景音乐

7. 根据 TTS 时间轴生成字幕（SRT 格式）

8. 输出 1080x1920 竖屏，H.264 编码，比特率 4M

9. 上传成品到 OSS

10. 回调 Proclaw 主后端更新任务状态

**3.12.3 性能要求**

- 单个任务处理时间 ≤ 5 分钟（含 TTS 调用）

- 支持至少 3 个 Worker 并发

- 任务超时时间：10 分钟

**3.12.4 接口定义**

```

POST /mix/submit

请求体: 任务参数

响应: { "task_id": "xxx", "status": "processing" }

GET /mix/status/{task_id}

响应: { "status": "completed", "video_url": "https://..." }

```
